## 2. MT (Message Type) 기초

### MT란?
```
MT = Message Type
   = SWIFT가 1970년대부터 사용해온 전통적 금융전문 표준
   = 태그(Tag) 기반의 텍스트 형식
   = FIN(Financial Application) 서비스로 전송
```

### MT 전문 구조
```
┌─────────────────────────────────────────────────────┐
│                  MT 전문 전체 구조                    │
├─────────────────────────────────────────────────────┤
│  {1: Basic Header Block        }  ← 기본 헤더        │
│  {2: Application Header Block  }  ← 애플리케이션 헤더 │
│  {3: User Header Block         }  ← 사용자 헤더       │
│  {4: Text Block (본문)         }  ← 실제 데이터 ★    │
│  {5: Trailer Block             }  ← 트레일러          │
└─────────────────────────────────────────────────────┘
```

### MT 전문 실제 예시 (MT103 - 송금)
```swift
{1:F01WOOBURKRAXXX0000000000}
{2:I103HNBKKRSEXXX}
{3:{108:OWP20240101001}}
{4:
:20:OWPREF240101001
:23B:CRED
:32A:240101KRW50000000,
:50K:/12345678901234
OVERSEAS BANK NEW YORK
123 WALL STREET, NEW YORK
:52A:CHASUS33
:53A:WOOBURKRXXX
:57A:HNBKKRSE
:59:/98765432109876
KOREAN IMPORT COMPANY
45 TEHERAN-RO, GANGNAM-GU, SEOUL
:70:OFFSHORE KRW PAYMENT /INV/2024-001
:71A:SHA
:72:/ACC/TRADE FINANCE
-}
{5:{CHK:ABCDEF123456}}
```

### MT 태그 명명 규칙
```
:20:   → 콜론(:) + 숫자 + 콜론(:) = 태그 식별자
:32A:  → A = 옵션 코드 (세부 형식 구분)
:50K:  → K = 옵션 코드

태그 번호 체계:
  20번대  → Reference 관련
  30번대  → 날짜 관련
  50번대  → 송금인(Ordering Customer) 관련
  56~58번 → 환거래/중개/수취 은행
  59번    → 수취인(Beneficiary)
  70번    → 내용/목적
  71번    → 수수료
```

## 3. MX (ISO 20022) 기초

### MX란?
```
MX  = Message eXchange (ISO 20022 기반 SWIFT 메시지)
    = 2004년 ISO(국제표준화기구)가 제정한 금융메시지 표준
    = XML(eXtensible Markup Language) 기반
    = 구조화된 데이터, 풍부한 정보 담기 가능
    = SWIFT의 미래 표준 ★

ISO 20022 = 금융 서비스용 전자 데이터 교환 표준
            은행, 증권, 외환, 카드 등 전 금융영역 포괄
```

## 4. MT vs MX 핵심 비교

### 4-1. 구조 비교

```
┌─────────────────────┬─────────────────────────────────────────┐
│        MT           │               MX (ISO 20022)            │
├─────────────────────┼─────────────────────────────────────────┤
│  태그 기반 텍스트    │  XML 구조화 데이터                        │
│                     │                                          │
│  :32A:240101        │  <IntrBkSttlmDt>                        │
│       KRW50000000,  │    2024-01-01                           │
│                     │  </IntrBkSttlmDt>                       │
│                     │  <IntrBkSttlmAmt Ccy="KRW">             │
│                     │    50000000                             │
│                     │  </IntrBkSttlmAmt>                      │
├─────────────────────┼─────────────────────────────────────────┤
│  사람이 읽기 쉬움    │  기계가 처리하기 쉬움                     │
│  (숙련자 기준)       │  (표준 XML 파서 사용 가능)                │
├─────────────────────┼─────────────────────────────────────────┤
│  전문 크기 작음      │  전문 크기 큼 (XML 태그 오버헤드)          │
├─────────────────────┼─────────────────────────────────────────┤
│  정보량 제한적       │  풍부한 정보 담기 가능                    │
│  (필드 길이 제한)    │  (구조화된 주소, LEI 등)                  │
└─────────────────────┴─────────────────────────────────────────┘
```

### 4-2. 데이터 표현 비교

```
항목별 MT vs MX 데이터 표현 방식

──────────────────────────────────────────────────────────────

[1] 금액 표현

  MT:   :32A:240101KRW50000000,
        └──날짜──┘└통화┘└──금액──┘
        → 날짜+통화+금액 모두 한 필드에 압축

  MX:   <IntrBkSttlmDt>2024-01-01</IntrBkSttlmDt>
        <IntrBkSttlmAmt Ccy="KRW">50000000</IntrBkSttlmAmt>
        → 날짜, 통화, 금액 각각 분리된 필드

──────────────────────────────────────────────────────────────

[2] 주소 표현

  MT:   :50K:/12345678901234
        OVERSEAS BANK NEW YORK
        123 WALL STREET NEW YORK USA
        → 자유형식 텍스트 (4줄 35자 제한)

  MX:   <Dbtr>
          <Nm>OVERSEAS BANK NEW YORK</Nm>
          <PstlAdr>
            <StrtNm>123 WALL STREET</StrtNm>
            <TwnNm>NEW YORK</TwnNm>
            <Ctry>US</Ctry>
          </PstlAdr>
        </Dbtr>
        → 구조화된 주소 (도시/국가 분리)

──────────────────────────────────────────────────────────────

[3] 거래 추적 ID

  MT:   :20:OWPREF240101001
        → 단순 Reference (16자 제한)

  MX:   <PmtId>
          <InstrId>OWPREF240101001</InstrId>
          <EndToEndId>E2E-OWP-2024-001</EndToEndId>
          <TxId>TXN-20240101-001</TxId>
          <UETR>a1b2c3d4-e5f6-7890-abcd-ef1234567890</UETR>
        </PmtId>
        → 다중 추적 ID (UETR = 글로벌 유일 거래 ID)

──────────────────────────────────────────────────────────────

[4] 법인 식별

  MT:   제한적 (BIC + 자유텍스트)

  MX:   <Id>
          <OrgId>
            <LEI>HWUPKR0MPOU8FGXBT394</LEI>
          </OrgId>
        </Id>
        → LEI (Legal Entity Identifier) 지원
```

### 4-3. 기술 스펙 비교

| 항목 | MT | MX (ISO 20022) |
|------|-----|-----------------|
| **데이터 형식** | 독자 태그 텍스트 | XML |
| **인코딩** | ASCII | UTF-8 |
| **스키마 검증** | SWIFT 규격서 수동 확인 | XSD 자동 검증 |
| **전문 크기** | 작음 (~수KB) | 큼 (~수십KB) |
| **필드 길이** | 엄격한 제한 | 유연함 |
| **다국어 지원** | 제한적 | UTF-8 완전 지원 |
| **버전 관리** | MT 버전 변경 어려움 | 버전별 XSD 관리 |
| **전송 채널** | SWIFT FIN | SWIFT SWIFTNet |
| **표준 관리** | SWIFT 독자 | ISO 국제표준 |
| **파싱 도구** | 전용 파서 필요 | 표준 XML 파서 사용 |

## 5. MT ↔ MX 메시지 대응표

### 결제(Payments) 영역

```
┌──────────────────┬────────────────┬─────────────────────────────┐
│   MT 전문        │   MX 전문      │         설명                 │
├──────────────────┼────────────────┼─────────────────────────────┤
│ MT 103           │ pacs.008       │ 고객 송금 (Customer Transfer)│
│ MT 202           │ pacs.009       │ 금융기관 송금 (FI Transfer)  │
│ MT 202 COV       │ pacs.009 COV   │ Cover Payment               │
│ MT 200           │ pacs.009       │ 자기앞 자금이체              │
│ MT 205           │ pacs.009       │ 3자 자금이체                 │
│ MT 900           │ camt.054       │ 출금 확인 (Debit Confirm)    │
│ MT 910           │ camt.054       │ 입금 확인 (Credit Confirm)   │
│ MT 940/941       │ camt.052       │ 계좌 보고 (Account Report)   │
│ MT 950           │ camt.053       │ 잔액 보고 (Statement)        │
│ MT 195           │ camt.056       │ 조회/취소 요청               │
│ MT 196           │ camt.029       │ 조회/취소 응답               │
│ MT 199           │ camt.028       │ 자유형식 조회                │
└──────────────────┴────────────────┴─────────────────────────────┘
```

### MX 메시지 명명 규칙 이해

```
pacs . 008 . 001 . 09
│       │     │    │
│       │     │    └── 버전 (Version)
│       │     └─────── 변형 (Variant)
│       └───────────── 메시지 번호
└───────────────────── 비즈니스 영역 약어

비즈니스 영역 약어:
  pacs = Payments Clearing and Settlement (결제/청산/정산)
  camt = Cash Management (현금 관리)
  pain = Payments Initiation (결제 개시)
  acmt = Account Management (계좌 관리)
  auth = Authorities (규제기관 보고)
  fxtr = Foreign Exchange Trade (외환거래)
```

### 주요 MX 메시지 상세

```
역외원화결제 관련 주요 MX 메시지

pacs.008  ─── 고객 송금
              MT103 대응
              해외거래은행 → 우리은행 → 한국은행

              우리은행 → 한국은행 자금이체

              MT950 대응
              한국은행 잔액 보고
              
camt.054  ─── 출금/입금 확인 통보

## 6. UETR - MX의 핵심 혁신

### UETR이란?
```
UETR = Unique End-to-End Transaction Reference
     = 전 세계 모든 결제를 추적하는 글로벌 고유 거래 ID
     = UUID v4 형식 (128비트 랜덤)
     = SWIFT gpi(global payments innovation)의 핵심

형식: xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx
예시: a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

### MT vs MX 거래 추적 비교

```
[MT 방식 - 기존]

은행A → 은행B → 은행C → 은행D
 :20:REF001   :20:REF002   :20:REF003
 
 ❌ 각 은행마다 다른 Reference 사용
 ❌ 중간에 Reference가 바뀌어 추적 단절
 ❌ 수취인이 "내 돈 어디있어?" 추적 불가

──────────────────────────────────────

[MX/gpi 방식 - 신규]

은행A → 은행B → 은행C → 은행D
 UETR: a1b2c3d4-...  (동일 UETR 전 구간 유지)
 
 ✅ 전 구간 동일 UETR로 추적 가능
 ✅ SWIFT gpi 추적기에서 실시간 조회
 ✅ 수취인도 직접 조회 가능
 ✅ 30분 이내 처리 SLA 측정 가능
```

### UETR 생성 코드

```python
import uuid
from datetime import datetime

def generate_uetr() -> str:
    """
    SWIFT gpi UETR 생성
    UUID v4 형식
    """
    return str(uuid.uuid4())

def generate_payment_ids(sender_ref: str) -> dict:
    """
    MX 결제용 ID 세트 생성
    """
    uetr = generate_uetr()
    timestamp = datetime.now().strftime("%Y%m%d%H%M%S")
    
    return {
        "InstrId"    : f"INSTR-{sender_ref}",       # 지시 ID
        "EndToEndId" : f"E2E-{sender_ref}",          # 종단간 ID
        "TxId"       : f"TXN-{timestamp}-{sender_ref[:8]}", # 거래 ID
        "UETR"       : uetr,                          # 글로벌 추적 ID ★
    }

# 사용 예시
ids = generate_payment_ids("OWP20240101001")
print(f"UETR: {ids['UETR']}")
# UETR: a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

## 7. 공존 기간(Coexistence Period) 이해

### 현재 상황 (2024)
```
┌─────────────────────────────────────────────────────────────┐
│                  MT/MX 공존 기간 타임라인                     │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  2022        2023        2024        2025        2026+      │
│   │           │           │           │           │          │
│   ▼           ▼           ▼           ▼           ▼          │
│  [공존        [MT 유지    [MT 유지    [MT 종료    [MX        │
│   시작]        MX 확산]    MX 주류]    예정]       완전전환]   │
│                                                              │
│  현재: MT와 MX 동시 사용                                      │
│  우리은행 프로젝트: MT 기반이나 MX 전환 고려 필요              │
└─────────────────────────────────────────────────────────────┘
```

### 변환(Translation) 처리

```
실무에서 MT ↔ MX 변환이 필요한 경우:

시나리오 1: 구형 거래은행 → 신형 거래은행
  [구형 은행]          [변환 게이트웨이]      [신형 은행]
  MT103 전송  ────────→  MT→MX 변환   ────→  pacs.008 수신

시나리오 2: 한국은행이 MX 전환 시
  [우리은행]           [한국은행]
  MT103 전송 ─────────→ MT→MX 자동변환 후 처리
  (현재 방식)           (미래 방식)

시나리오 3: 동일 전문 양방향 지원
  [우리은행 시스템]
  MT103  ←──── 수신 가능
  pacs.008 ←── 수신 가능   (동시 지원 필요)
```

### 변환 시 데이터 손실 주의

```
⚠️  MT → MX 변환 시 주의 필드

[문제 1] 주소 필드
  MT  :50K: 자유형식 35자 × 4줄
  → MX 구조화 주소 변환 시 파싱 오류 가능

  MT 원본:
  :50K:/123456789
  JOHN DOE C/O ABC COMPANY
  123 MAIN ST APT 4B NYC
  
  MX 변환 후 (어떤 필드에 넣어야 하나?):
  <Nm>JOHN DOE C/O ABC COMPANY</Nm>
  <StrtNm>???</StrtNm>  ← 파싱 어려움

[문제 2] 수수료 코드
  MT :71A: BEN/OUR/SHA (3가지)
  MX ChrgBr: DEBT/CRED/SHAR/SLEV (4가지)
  
  매핑:
  MT BEN → MX CRED
  MT OUR → MX DEBT
  MT SHA → MX SHAR
  (SLEV는 MX 신규 추가)

[문제 3] 길이 제한
  MT :70: 140자 (4줄 × 35자)
  MX <Ustrd> 이론상 무제한
  → MX→MT 변환 시 잘릴 수 있음 ⚠️
```

### MT 파서 구현

```python
import re
from typing import Optional

class MTMessageParser:
    """
    SWIFT MT 전문 파서
    태그 기반 텍스트 파싱
    """
    
    # Block 패턴
    BLOCK_PATTERN = re.compile(r'\{(\d):(.*?)\}(?=\{|\Z)', re.DOTALL)
    # 태그 패턴  
    TAG_PATTERN   = re.compile(r':(\w{2,3}):(.*?)(?=:\w{2,3}:|$)', re.DOTALL)
    
    def parse(self, raw_message: str) -> dict:
        """MT 전문 전체 파싱"""
        
        blocks = self._parse_blocks(raw_message)
        
        result = {
            "block1" : self._parse_block1(blocks.get("1", "")),
            "block2" : self._parse_block2(blocks.get("2", "")),
            "block3" : self._parse_block3(blocks.get("3", "")),
            "block4" : self._parse_block4(blocks.get("4", "")),
            "block5" : self._parse_block5(blocks.get("5", "")),
        }
        
        return result
    
    def _parse_blocks(self, raw: str) -> dict:
        """Block 1~5 분리"""
        blocks = {}
        for match in self.BLOCK_PATTERN.finditer(raw):
            block_num = match.group(1)
            block_content = match.group(2)
            blocks[block_num] = block_content
        return blocks
    
    def _parse_block4(self, block4_content: str) -> dict:
        """Block4 (본문) 태그 파싱"""
        tags = {}
        
        for match in self.TAG_PATTERN.finditer(block4_content):
            tag   = match.group(1)  # 예: "32A"
            value = match.group(2).strip()
            tags[tag] = value
        
        return tags
    
    def parse_32a(self, value: str) -> dict:
        """
        :32A: 태그 파싱
        형식: YYMMDDCCCAMOUNT
        예시: 240101KRW50000000,
        """
        pattern = re.compile(r'(\d{6})([A-Z]{3})(\d+,?\d*)')
        match = pattern.match(value)
        
        if not match:
            raise ValueError(f"잘못된 :32A: 형식: {value}")
        
        date_str = match.group(1)   # 240101
        currency = match.group(2)   # KRW
        amount   = match.group(3).replace(',', '.')  # 50000000
        
        return {
            "value_date" : f"20{date_str[:2]}-{date_str[2:4]}-{date_str[4:6]}",
            "currency"   : currency,
            "amount"     : float(amount),
        }
    
    def parse_50k(self, value: str) -> dict:
        """
        :50K: 태그 파싱 (Ordering Customer)
        형식: /계좌번호\n이름\n주소
        """
        lines = value.strip().split('\n')
        account = lines[0].lstrip('/') if lines[0].startswith('/') else ""
        name    = lines[1] if len(lines) > 1 else ""
        address = '\n'.join(lines[2:]) if len(lines) > 2 else ""
        
        return {
            "account" : account,
            "name"    : name,
            "address" : address,
        }


# ── MT 파서 사용 예시 ─────────────────────────────────────

{2:I103HNBKKRSEXXX}

### MX 파서 구현

```python
import xml.etree.ElementTree as ET
from decimal import Decimal
from typing import Optional

class MXMessageParser:
    """
    SWIFT MX (ISO 20022) 전문 파서
    XML 기반 파싱
    """
    
    # ISO 20022 네임스페이스
    NAMESPACES = {
        "pacs008" : "urn:iso:std:iso:20022:tech:xsd:pacs.008.001.09",
        "pacs009" : "urn:iso:std:iso:20022:tech:xsd:pacs.009.001.09",
        "camt053" : "urn:iso:std:iso:20022:tech:xsd:camt.053.001.09",
        "camt054" : "urn:iso:std:iso:20022:tech:xsd:camt.054.001.09",
    }
    
    def parse_pacs008(self, xml_str: str) -> dict:
        """
        pacs.008 파싱 (MT103 대응)
        FI-to-FI Customer Credit Transfer
        """
        root = ET.fromstring(xml_str)
        ns   = {"ns": self.NAMESPACES["pacs008"]}
        
        # 그룹 헤더
        grp_hdr = root.find(".//ns:GrpHdr", ns)
        
        # 거래 정보
        tx_info = root.find(".//ns:CdtTrfTxInf", ns)
        
        result = {
            "message_id"  : self._find_text(grp_hdr, "ns:MsgId", ns),
            "created_dt"  : self._find_text(grp_hdr, "ns:CreDtTm", ns),
            "num_of_txns" : self._find_text(grp_hdr, "ns:NbOfTxs", ns),
            
            "payment_id"  : {
                "instr_id"     : self._find_text(tx_info, ".//ns:InstrId", ns),
                "end_to_end_id": self._find_text(tx_info, ".//ns:EndToEndId", ns),
                "tx_id"        : self._find_text(tx_info, ".//ns:TxId", ns),
                "uetr"         : self._find_text(tx_info, ".//ns:UETR", ns),
            },
            
            "amount"      : {
                "currency" : self._find_attr(
                                tx_info, 
                                ".//ns:IntrBkSttlmAmt", 
                                "Ccy", ns),
                "value"    : Decimal(
                                self._find_text(
                                    tx_info, 
                                    ".//ns:IntrBkSttlmAmt", ns) or "0"),
                "sttlm_dt" : self._find_text(
                                tx_info, 
                                ".//ns:IntrBkSttlmDt", ns),
            },
            
            "charge_bearer" : self._find_text(tx_info, ".//ns:ChrgBr", ns),
            
            "debtor"      : {
                "name"    : self._find_text(tx_info, ".//ns:Dbtr/ns:Nm", ns),
                "country" : self._find_text(
                                tx_info, 
                                ".//ns:Dbtr/ns:PstlAdr/ns:Ctry", ns),
                "account" : self._find_text(
                                tx_info, 
                                ".//ns:DbtrAcct//ns:Id", ns),
                "bic"     : self._find_text(
                                tx_info, 
                                ".//ns:DbtrAgt//ns:BICFI", ns),
            },
            
            "creditor"    : {
                "name"    : self._find_text(tx_info, ".//ns:Cdtr/ns:Nm", ns),
                "country" : self._find_text(
                                tx_info, 
                                ".//ns:Cdtr/ns:PstlAdr/ns:Ctry", ns),
                "account" : self._find_text(
                                tx_info, 
                                ".//ns:CdtrAcct//ns:Id", ns),
                "bic"     : self._find_text(
                                tx_info, 
                                ".//ns:CdtrAgt//ns:BICFI", ns),
            },
            
            "remittance_info" : self._find_text(
                                    tx_info, 
                                    ".//ns:RmtInf/ns:Ustrd", ns),
        }
        
        return result
    
    def _find_text(self, 
                   element, 
                   path: str, 
                   ns: dict) -> Optional[str]:
        """XML 요소에서 텍스트 안전 추출"""
        if element is None:
            return None
        found = element.find(path, ns)
        return found.text if found is not None else None
    
    def _find_attr(self, 
                   element, 
                   path: str, 
                   attr: str, 
                   ns: dict) -> Optional[str]:
        """XML 요소에서 속성 안전 추출"""
        if element is None:
            return None
        found = element.find(path, ns)
        return found.get(attr) if found is not None else None

## 9. MT ↔ MX 변환기 구현

```python
from decimal import Decimal
from datetime import datetime

class MTtoMXConverter:
    """
    MT → MX 변환기
    MT103 → pacs.008
    """
    
    # 수수료 코드 매핑
    CHARGE_BEARER_MAP = {
        "BEN" : "CRED",   # 수취인 부담
        "OUR" : "DEBT",   # 송금인 부담
        "SHA" : "SHAR",   # 분담
    }
    
    def mt103_to_pacs008(self, mt103_parsed: dict) -> str:
        """MT103 → pacs.008 XML 변환"""
        
        block4 = mt103_parsed["block4"]
        
        # :32A: 파싱
        amount_info  = self._parse_32a(block4.get("32A", ""))
        
        # :50K: or :50F: 파싱
        debtor_info  = self._parse_ordering_customer(
                            block4.get("50K") or block4.get("50F", ""))
        
        # :59: or :59F: 파싱
        creditor_info = self._parse_beneficiary(
                            block4.get("59") or block4.get("59F", ""))
        
        # 수수료 코드 변환
        charge_bearer = self.CHARGE_BEARER_MAP.get(
                            block4.get("71A", "SHA"), "SHAR")
        
        # UETR 신규 생성 (MT에는 없는 개념)
        import uuid
        uetr = str(uuid.uuid4())
        
        xml = f"""<?xml version="1.0" encoding="UTF-8"?>
<Document xmlns="urn:iso:std:iso:20022:tech:xsd:pacs.008.001.09">
  <FIToFICstmrCdtTrf>
    <GrpHdr>
      <MsgId>{block4.get('20', '')}</MsgId>
      <CreDtTm>{datetime.now().strftime('%Y-%m-%dT%H:%M:%S')}</CreDtTm>
      <NbOfTxs>1</NbOfTxs>
      <SttlmInf>
        <SttlmMtd>CLRG</SttlmMtd>
      </SttlmInf>
    </GrpHdr>
    <CdtTrfTxInf>
      <PmtId>
        <InstrId>{block4.get('20', '')}</InstrId>
        <EndToEndId>{block4.get('20', '')}</EndToEndId>
        <UETR>{uetr}</UETR>
      </PmtId>
      <IntrBkSttlmAmt Ccy="{amount_info['currency']}">\
{amount_info['amount']}</IntrBkSttlmAmt>
      <IntrBkSttlmDt>{amount_info['value_date']}</IntrBkSttlmDt>
      <ChrgBr>{charge_bearer}</ChrgBr>
      <Dbtr>
        <Nm>{debtor_info.get('name', '')}</Nm>
      </Dbtr>
      <DbtrAcct>
        <Id><Othr><Id>{debtor_info.get('account', '')}</Id></Othr></Id>
      </DbtrAcct>
      <DbtrAgt>
        <FinInstnId>
          <BICFI>{block4.get('52A', '')}</BICFI>
        </FinInstnId>
      </DbtrAgt>
      <CdtrAgt>
        <FinInstnId>
          <BICFI>{block4.get('57A', '')}</BICFI>
        </FinInstnId>
      </CdtrAgt>
      <Cdtr>
        <Nm>{creditor_info.get('name', '')}</Nm>
      </Cdtr>
      <CdtrAcct>
        <Id><Othr><Id>{creditor_info.get('account', '')}</Id></Othr></Id>
      </CdtrAcct>
      <RmtInf>
        <Ustrd>{block4.get('70', '')}</Ustrd>
      </RmtInf>
    </CdtTrfTxInf>
  </FIToFICstmrCdtTrf>
</Document>"""
        
        return xml
    
    def _parse_32a(self, value: str) -> dict:
        import re
        pattern = re.compile(r'(\d{6})([A-Z]{3})(\d+,?\d*)')
        match = pattern.match(value)
        if not match:
            return {"currency": "", "amount": "0", "value_date": ""}
        
        date_str = match.group(1)
        return {
            "value_date" : f"20{date_str[:2]}-{date_str[2:4]}-{date_str[4:6]}",
            "currency"   : match.group(2),
            "amount"     : match.group(3).replace(',', '.'),
        }
    
    def _parse_ordering_customer(self, value: str) -> dict:
        lines = value.strip().split('\n')
        account = lines[0].lstrip('/') if lines and lines[0].startswith('/') else ""
        name    = lines[1].strip() if len(lines) > 1 else ""
        return {"account": account, "name": name}
    
    def _parse_beneficiary(self, value: str) -> dict:
        lines = value.strip().split('\n')
        account = lines[0].lstrip('/') if lines and lines[0].startswith('/') else ""
        name    = lines[1].strip() if len(lines) > 1 else ""
        return {"account": account, "name": name}

## 12. 실습 체크리스트

```
📝 MT vs MX 이해 확인 체크리스트

기초 이해
  □ MT103과 pacs.008이 동일한 기능임을 설명할 수 있다
  □ BIC 필드가 MT(:57A:)와 MX(<BICFI>)에서 다르게 표현됨을 안다
  □ UETR이 MT에는 없고 MX에서 도입된 개념임을 안다
  □ 공존 기간(Coexistence)의 의미를 설명할 수 있다

파싱 구현
  □ MT103 원문을 보고 태그별로 의미를 읽을 수 있다
  □ pacs.008 XML을 보고 필드별 의미를 읽을 수 있다
  □ :32A: 필드를 날짜/통화/금액으로 분리할 수 있다
  □ XML 네임스페이스를 이해하고 파싱에 적용할 수 있다

변환 이해
  □ MT SHA → MX SHAR 수수료 코드 매핑을 안다
  □ MT 자유형식 주소 → MX 구조화 주소 변환 문제를 안다
  □ MT→MX 변환 시 UETR을 신규 생성해야 함을 안다

역외원화 적용
  □ 현재 프로젝트가 MT 기반임을 이해한다
  □ 향후 MX 전환을 대비한 추상화 설계를 이해한다
  □ 한국은행 연동 전문 MT→MX 대응표를 외울 수 있다
```
