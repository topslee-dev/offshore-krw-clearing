## 2. MT103 : Customer Credit Transfer

### 2-1. MT103이란?

```
MT103 = 고객 송금 전문
      = 은행이 고객을 대신하여 다른 은행 고객에게 자금 이체
      = 역외원화결제에서 가장 핵심 전문 ★

사용 시나리오:
  ① 해외기업 → 우리은행 → 한국기업 (외화 → 원화 송금)
  ② 우리은행 → 한국은행  (BOK 통보용 MT103)
  ③ 국내기업 → 우리은행 → 해외기업 (원화 → 외화 송금)
```

### 2-3. MT103 필드 완전 해설

#### :20: Sender's Reference (발신 참조번호)

```
형식   : 16자 이내 (영숫자, /, -)
필수   : M (Mandatory)
예시   : OWPREF240101001

┌──────────────────────────────────────────────────────────┐
│  :20: 상세 규칙                                            │
├──────────────────────────────────────────────────────────┤
│  허용문자 : A-Z, a-z, 0-9, /, -                           │
│  금지문자 : 특수문자 대부분, 공백으로 시작/끝 불가           │
│  최대길이 : 16자                                           │
│  이중슬래시(//) 금지                                       │
│  슬래시(/)로 시작/끝 금지                                   │
├──────────────────────────────────────────────────────────┤
│  역외원화결제 Reference 설계 예시                           │
│  OWP + YYYYMMDD + 일련번호(5자리)                          │
│  → OWP2024010100001                                       │
│                                                            │
│  중요: 동일 Reference 중복 발송 금지 (멱등성)               │
└──────────────────────────────────────────────────────────┘
```

#### :13C: Time Indication (시각 표시)

```
형식   : /코드/HHMM+/-IINN
필수   : O (Optional)
예시   : /CLSTIME/1400+0900

코드 종류:
  /CLSTIME/ = Closing Time (결제 마감시각)
  /RNCTIME/ = Reconciliation Time (대사 시각)
  /SNDTIME/ = Sending Time (발신 시각)

시간대 표시:
  +0900 = UTC+9 (한국 표준시 KST)
  +0000 = UTC
  -0500 = EST (미국 동부)

역외원화 포인트:
  ✅ 한국은행 마감 시각 기재 권장
  ✅ KST 기준으로 기재 (UTC+0900)
```

#### :23B: Bank Operation Code (은행 업무 코드)

```
형식   : 4자리 영문자
필수   : M (Mandatory)
예시   : CRED

코드 목록:
┌────────┬─────────────────────────────────────────────────┐
│  코드  │  설명                                             │
├────────┼─────────────────────────────────────────────────┤
│ CRED   │ 일반 송금 (Credit Transfer) ← 가장 일반적 ★      │
│ CRTS   │ Treasury 관련 송금                               │
│ SPAY   │ SWIFT Pay                                        │
│ SSTD   │ SWIFT Standard                                   │
└────────┴─────────────────────────────────────────────────┘

역외원화결제 → 항상 CRED 사용
```

#### :23E: Instruction Code (지시 코드)

```
형식   : 4자리 코드 [/추가정보]
필수   : O (Optional)
예시   : HOLD / PHON / REPA

코드 목록:
┌────────┬──────────────────────────────────────────────────┐
│  코드  │  설명                                              │
├────────┼──────────────────────────────────────────────────┤
│ HOLD   │ 수취인 내방 시 지급 (Hold for Beneficiary)         │
│ PHON   │ 전화 확인 후 지급 (Phone Before Payment)           │
│ TELB   │ 전화 확인 후 지급 (대체 코드)                       │
│ REPA   │ Repair 요청                                       │
│ CHQB   │ 수표 발행 후 우편 발송                              │
│ PBEN   │ 지급 전 수취인 연락                                 │
│ URGP   │ 긴급 처리 요청                                     │
└────────┴──────────────────────────────────────────────────┘

복수 사용 가능 (최대 2개)
:23E:HOLD
:23E:PHON
```

#### :26T: Transaction Type Code (거래 유형 코드)

```
형식   : 3자리 영숫자
필수   : O (Optional)
예시   : K90

주요 코드 (국제 표준):
┌─────┬──────────────────────────────────────────────────────┐
│코드 │ 설명                                                   │
├─────┼──────────────────────────────────────────────────────┤
│ K90 │ 무역대금 결제 (Trade Payment)                          │
│ K02 │ 금융거래 (Financial Transaction)                       │
│ K03 │ 용역대금 (Service Payment)                             │
│ K04 │ 투자 (Investment)                                      │
│ K05 │ 배당금 (Dividend)                                      │
└─────┴──────────────────────────────────────────────────────┘

역외원화결제:
  ✅ 무역대금 → K90
  ✅ :77B: 외환규제 보고와 연계
```

#### :32A: Value Date / Currency / Amount (결제일/통화/금액)

```
형식   : YYMMDD + CCY(3) + Amount
필수   : M (Mandatory)
예시   : 240101KRW50000000,

┌──────────────────────────────────────────────────────────┐
│  :32A: 필드 분해                                           │
├──────────────────────────────────────────────────────────┤
│  24   01   01   KRW   50000000,                          │
│  └─┘  └─┘  └─┘  └──┘  └──────┘                          │
│  연   월   일   통화   금액                                 │
├──────────────────────────────────────────────────────────┤
│  금액 표기 규칙                                             │
│  ① 소수점 = 콤마(,) 사용                                   │
│     50000000,    → 50,000,000 원 (정수)                   │
│     38461,54     → 38,461.54 달러 (소수)                  │
│  ② 천단위 구분자 없음                                       │
│  ③ 최대 15자리                                             │
├──────────────────────────────────────────────────────────┤
│  역외원화결제 주의사항                                       │
│  ✅ 통화코드 = KRW (원화)                                   │
│  ✅ 원화는 소수점 없음 → 50000000, (콤마로 끝)              │
│  ✅ Value Date = 실제 자금 이동일 (영업일 기준)              │
│  ⚠️  주말/공휴일 Value Date 설정 오류 주의                  │
└──────────────────────────────────────────────────────────┘
```

#### :33B: Currency/Instructed Amount (지시 통화/금액)

```
형식   : CCY(3) + Amount
필수   : O (Optional)
예시   : USD38461,54

사용 목적:
  :32A: 와 :33B: 통화가 다를 때 사용
  → :33B: = 원래 지시된 외화 금액
  → :32A: = 환전 후 실제 결제 금액

예시:
  :32A:240101KRW50000000,   ← 실제 결제: 5천만원
  :33B:USD38461,54           ← 원래 지시: 38,461.54 달러
  :36:1,3                    ← 환율: 1,300원/달러

역외원화 시나리오:
  해외기업이 USD로 지시 → 우리은행이 KRW로 환전 후 결제
  → :33B: = USD 원금
  → :32A: = 환전된 KRW 금액
  → :36:  = 적용 환율
```

#### :36: Exchange Rate (환율)

```
형식   : 소수 (콤마 구분)
필수   : O (Optional, :33B: 있을 때 권장)
예시   : 1300, (1달러 = 1,300원)

표기 방법:
  1,3     → 1.3
  1300,   → 1300.0 (1달러 = 1,300원)
  0,0008  → 0.0008 (역환율)

역외원화 계산 예시:
  USD 38,461.54 × 환율 1,300 = KRW 50,000,002 ≈ 50,000,000
  :33B:USD38461,54
  :36:1300,
  :32A:240101KRW50000000,
```

#### :51A: Sending Institution (대행 발신기관)

```
형식   : [/계좌] BIC
필수   : O (Optional)
예시   : :51A:CHASUS33

사용 목적:
  다른 은행을 대신하여 전문을 발신할 때
  → 대행 발신기관의 BIC 기재

역외원화결제: 일반적으로 미사용
```

#### :52A: / :52D: Ordering Institution (송금의뢰 은행)

```
형식   : [/계좌] BIC  (:52A:)
         [/계좌] 이름/주소 (:52D:)
필수   : O (Optional)
예시   : :52A:CHASUS33

사용 목적:
  :50K: 의 고객 계좌를 관리하는 은행
  해외에서 우리은행으로 송금 시 → 해외거래은행 BIC

역외원화결제 예시:
  해외기업(JP Morgan 고객)이 송금
  :50K: JP Morgan 고객 정보
  :52A: CHASUS33 (JP Morgan)
```

#### :53A: / :53B: / :53D: Sender's Correspondent (발신 환거래은행)

```
형식   : [/계좌] BIC  (:53A:)
         [/계좌][위치] (:53B:)
         [/계좌] 이름/주소 (:53D:)
필수   : O (Optional)
예시   : :53A:WOOBURKRXXX

사용 목적:
  발신은행(Sender)의 환거래은행
  실제 자금이 출금되는 노스트로 계좌 보유 은행

역외원화결제 플로우:
  해외거래은행 → 우리은행(환거래) → 한국은행
  :53A: = WOOBURKRXXX (우리은행)
  → 우리은행 계좌에서 자금 출금
```

#### :54A: / :54B: / :54D: Receiver's Correspondent (수신 환거래은행)

```
형식   : [/계좌] BIC  (:54A:)
필수   : O (Optional)
예시   : :54A:HNBKKRSEXXX

사용 목적:
  수신은행(Receiver)의 환거래은행
  자금이 최종 입금되는 은행

역외원화결제:
  :54A: = HNBKKRSEXXX (한국은행)
  → 한국은행 계좌로 자금 입금
```

```
형식   : [/계좌] BIC

  3자 정산이 필요한 경우

#### :56A: / :56C: / :56D: Intermediary (중개기관)

```
형식   : [/계좌] BIC  (:56A:)
         /계좌번호     (:56C:)
         [/계좌] 이름  (:56D:)
필수   : O (Optional)
예시   : :56A:HVBKKRSEXXX

사용 목적:
  수취은행(:57A:)에 도달하기 위한 경유 은행
  직접 연결이 없을 때 중간 경유 은행 지정

역외원화결제:
  우리은행 ↔ 한국은행 직접 연결 → 일반적으로 미사용
  특수 경로 필요 시 사용
```

#### :57A: / :57B: / :57C: / :57D: Account With Institution (계좌관리기관)

```
형식   : [/계좌] BIC  (:57A:)  ← 가장 일반적
         [/계좌][위치] (:57B:)
         /계좌번호     (:57C:)
         [/계좌] 이름  (:57D:)
필수   : O (Optional, 실무상 필수에 가까움)
예시   : :57A:HNBKKRSE

사용 목적:
  수취인(:59:)의 계좌가 개설된 금융기관
  ★ 역외원화결제에서 핵심 필드

역외원화결제:
  :57A:HNBKKRSE  → 한국은행 ★★★
  수취인 계좌가 한국은행에 개설된 경우

:57D: 사용 예시 (BIC + 이름 + 주소):
  :57D:/KR12345678
  BANK OF KOREA
  39 NAMDAEMUN-RO JUNG-GU SEOUL
```

│  :59: (자유텍스트)    │  :59F: (구조화)                   │
├──────────────────────┼───────────────────────────────────┤
│  사람이 읽기 쉬움     │  기계 처리 용이                    │
│  STP 실패 가능       │  STP 처리 유리                     │
│  파싱 어려움         │  파싱 명확                         │
│  레거시 시스템 사용   │  신규 시스템 권장                  │
└──────────────────────┴───────────────────────────────────┘
```

#### :70: Remittance Information (송금 목적/내용)

```
형식   : 최대 4줄 × 35자 = 140자
필수   : O (Optional)
예시   : /INV/2024-001
         OFFSHORE KRW PAYMENT
         TRADE FINANCE SETTLEMENT

구조화 코드:
  /INV/  = Invoice 번호
  /IPI/  = International Payment Instruction
  /RFB/  = Reference for Beneficiary
  /ROC/  = Reference of Customer

역외원화결제 포인트:
  ✅ 거래 목적 명확히 기재
  ✅ 인보이스 번호 포함 (/INV/)
  ✅ 한국은행 외환규제 보고용 정보 포함
  ⚠️  140자 초과 불가
```

#### :71A: Details of Charges (수수료 부담 방식)

```
형식   : 3자리 코드
필수   : M (Mandatory)
예시   : SHA

코드 목록:
┌──────┬──────────────┬─────────────────────────────────────┐
│ 코드 │ 영문명        │ 설명                                 │
├──────┼──────────────┼─────────────────────────────────────┤
│ BEN  │ Beneficiary  │ 수수료 전부 수취인 부담               │
│      │              │ → 수취인이 수수료 차감 후 수령        │
├──────┼──────────────┼─────────────────────────────────────┤
│ OUR  │ Ordering     │ 수수료 전부 송금인 부담               │
│      │              │ → 수취인이 전액 수령                 │
│      │              │ → :71F:, :71G: 함께 사용             │
├──────┼──────────────┼─────────────────────────────────────┤
│ SHA  │ Shared       │ 수수료 분담 (가장 일반적)             │
│      │              │ → 송금측: 자국 수수료 부담            │
│      │              │ → 수취측: 상대국 수수료 부담          │
└──────┴──────────────┴─────────────────────────────────────┘

MX 대응:
  BEN → CRED
  OUR → DEBT
  SHA → SHAR
```

#### :71F: Sender's Charges (송금은행 수수료)

```
형식   : CCY(3) + Amount
필수   : O (Optional, :71A:=OUR 일 때 사용)
예시   : :71F:USD10,

:71A:OUR 일 때:
  → 송금인이 모든 수수료 부담
  → 발신 은행 수수료를 :71F: 에 기재
  → 중개 은행 수수료도 별도 :71F: 추가 가능
```

#### :71G: Receiver's Charges (수취은행 수수료)

```
형식   : CCY(3) + Amount
필수   : O (Optional, :71A:=OUR 일 때 사용)
예시   : :71G:KRW5000,

:71A:OUR 일 때:
  → 수취 은행 수수료를 :71G: 에 기재
  → :32A: 금액에서 :71G: 차감 후 수취인 지급
```

#### :72: Sender to Receiver Information (은행간 참고사항)

```
형식   : 최대 6줄 × 35자
필수   : O (Optional)
예시   : /ACC/TRADE FINANCE
         /INS/PAYMENT ON BEHALF
         /BNF/SEE REMITTANCE INFO

구조화 코드:
  /ACC/  = 계좌 관련 지시
  /INS/  = 지시사항 (Instruction)
  /INT/  = 중개기관 정보
  /BNF/  = 수취인 관련 정보
  /REC/  = 수신은행 관련 정보
  /REJT/ = 거절 이유
  /RETN/ = 반환 이유

역외원화결제:
  /ACC/ 로 계좌 관련 특이사항 기재
  /INS/ 로 한국은행 처리 지시사항 기재
```

```
형식   : 최대 3줄 × 35자
필수   : O (Optional, 외환규제 시 필수)

역외원화결제:

## 3. MT103 파싱 및 생성 코드

```python
import re
from decimal import Decimal
from datetime import date, datetime
from dataclasses import dataclass, field
from typing import Optional, Dict, List

@dataclass
class CustomerInfo:
    """송금인/수취인 정보"""
    account : str = ""
    name    : str = ""
    address : List[str] = field(default_factory=list)
    country : str = ""
    city    : str = ""
    bic     : str = ""

@dataclass
class MT103:
    """MT103 도메인 객체"""
    # 필수 필드
    sender_ref       : str = ""
    bank_op_code     : str = "CRED"
    value_date       : Optional[date] = None
    currency         : str = "KRW"
    amount           : Optional[Decimal] = None

    # 선택 필드
    time_indication  : str = ""
    instruction_code : str = ""
    transaction_type : str = ""
    instructed_ccy   : str = ""
    instructed_amt   : Optional[Decimal] = None
    exchange_rate    : Optional[Decimal] = None

    # 당사자
    ordering_customer    : CustomerInfo = field(default_factory=CustomerInfo)
    sending_institution  : str = ""
    ordering_institution : str = ""
    senders_correspondent: str = ""
    receivers_corr       : str = ""
    third_reimb_inst     : str = ""
    intermediary         : str = ""
    account_with_inst    : str = ""
    beneficiary          : CustomerInfo = field(default_factory=CustomerInfo)

    # 부가정보
    remittance_info      : str = ""
    charge_bearer        : str = "SHA"
    senders_charges      : str = ""
    receivers_charges    : str = ""
    sender_to_receiver   : str = ""
    regulatory_reporting : str = ""


class MT103Parser:
    """
    MT103 전문 파서 및 생성기
    역외원화결제 특화
    """

    TAG_PATTERN = re.compile(
        r':(\w{2,3}):(.*?)(?=\n:\w{2,3}:|\n-}|$)',
        re.DOTALL
    )
    BLOCK4_PATTERN = re.compile(
        r'\{4:\s*\n(.*?)\n-\}',
        re.DOTALL
    )
    AMOUNT_32A = re.compile(
        r'(?P<date>\d{6})(?P<ccy>[A-Z]{3})(?P<amount>[\d,]+)'
    )
    AMOUNT_33B = re.compile(
        r'(?P<ccy>[A-Z]{3})(?P<amount>[\d,]+)'
    )

    def parse(self, raw: str) -> MT103:
        """MT103 원문 파싱"""

        tags = self._extract_tags(raw)
        mt   = MT103()

        # ── 필수 필드 ──────────────────────────────
        mt.sender_ref   = tags.get("20", "").strip()
        mt.bank_op_code = tags.get("23B", "CRED").strip()

        # :32A: 파싱
        if "32A" in tags:
            parsed_32a = self._parse_32a(tags["32A"])
            mt.value_date = parsed_32a["date"]
            mt.currency   = parsed_32a["currency"]
            mt.amount     = parsed_32a["amount"]

        # ── 선택 필드 ──────────────────────────────
        mt.time_indication  = tags.get("13C", "").strip()
        mt.instruction_code = tags.get("23E", "").strip()
        mt.transaction_type = tags.get("26T", "").strip()

        # :33B: 파싱
        if "33B" in tags:
            parsed_33b = self._parse_33b(tags["33B"])
            mt.instructed_ccy = parsed_33b["currency"]
            mt.instructed_amt = parsed_33b["amount"]

        # :36: 환율
        if "36" in tags:
            rate_str = tags["36"].strip().replace(",", ".")
            mt.exchange_rate = Decimal(rate_str)

        # ── 당사자 정보 ────────────────────────────
        # 송금의뢰인
        for tag in ["50F", "50K", "50A"]:
            if tag in tags:
                mt.ordering_customer = self._parse_customer(
                                            tags[tag], tag[-1])
                break

        mt.sending_institution   = tags.get("51A", "").strip()
        mt.ordering_institution  = tags.get("52A", "").strip()
        mt.senders_correspondent = tags.get("53A", "").strip()
        mt.receivers_corr        = tags.get("54A", "").strip()
        mt.third_reimb_inst      = tags.get("55A", "").strip()
        mt.intermediary          = tags.get("56A", "").strip()
        mt.account_with_inst     = (tags.get("57A") or
                                    tags.get("57D") or
                                    tags.get("57B") or "").strip()

        # 수취인
        for tag in ["59F", "59A", "59"]:
            if tag in tags:
                opt = tag[2:] if len(tag) == 3 else ""
                mt.beneficiary = self._parse_customer(tags[tag], opt)
                break

        # ── 부가정보 ────────────────────────────────
        mt.remittance_info    = tags.get("70", "").strip()
        mt.charge_bearer      = tags.get("71A", "SHA").strip()
        mt.senders_charges    = tags.get("71F", "").strip()
        mt.receivers_charges  = tags.get("71G", "").strip()
        mt.sender_to_receiver = tags.get("72", "").strip()
        mt.regulatory_reporting = tags.get("77B", "").strip()

        return mt

    def build(self, mt: MT103) -> str:
        """MT103 도메인 객체 → 원문 생성"""

        lines = ["{4:"]

        # 필수 태그
        lines.append(f":20:{mt.sender_ref}")

        if mt.time_indication:
            lines.append(f":13C:{mt.time_indication}")

        lines.append(f":23B:{mt.bank_op_code}")

        if mt.instruction_code:
            lines.append(f":23E:{mt.instruction_code}")

        if mt.transaction_type:
            lines.append(f":26T:{mt.transaction_type}")

        # :32A: 생성
        date_str   = mt.value_date.strftime("%y%m%d")
        amount_str = str(mt.amount).replace(".", ",")
        lines.append(f":32A:{date_str}{mt.currency}{amount_str}")

        # :33B: 환전 정보
        if mt.instructed_amt and mt.instructed_ccy:
            amt = str(mt.instructed_amt).replace(".", ",")
            lines.append(f":33B:{mt.instructed_ccy}{amt}")

        if mt.exchange_rate:
            rate = str(mt.exchange_rate).replace(".", ",")
            lines.append(f":36:{rate}")

        # 송금의뢰인 (:50F: 구조화 방식 우선)
        oc = mt.ordering_customer
        lines.append(f":50F:/{oc.account}")
        lines.append(f"1/{oc.name}")
        if oc.address:
            lines.append(f"2/{oc.address[0]}")
        if oc.country and oc.city:
            lines.append(f"3/{oc.country}/{oc.city}")

        # 선택 기관 필드
        for tag, val in [
            ("51A", mt.sending_institution),
            ("52A", mt.ordering_institution),
            ("53A", mt.senders_correspondent),
            ("54A", mt.receivers_corr),
            ("55A", mt.third_reimb_inst),
            ("56A", mt.intermediary),
            ("57A", mt.account_with_inst),
        ]:
            if val:
                lines.append(f":{tag}:{val}")

        # 수취인 (:59F: 구조화 방식)
        bc = mt.beneficiary
        lines.append(f":59F:/{bc.account}")
        lines.append(f"1/{bc.name}")
        if bc.address:
            lines.append(f"2/{bc.address[0]}")
        if bc.country and bc.city:
            lines.append(f"3/{bc.country}/{bc.city}")

        # 부가 정보
        if mt.remittance_info:
            lines.append(f":70:{mt.remittance_info}")

        lines.append(f":71A:{mt.charge_bearer}")

        if mt.senders_charges:
            lines.append(f":71F:{mt.senders_charges}")
        if mt.receivers_charges:
            lines.append(f":71G:{mt.receivers_charges}")
        if mt.sender_to_receiver:
            lines.append(f":72:{mt.sender_to_receiver}")
        if mt.regulatory_reporting:
            lines.append(f":77B:{mt.regulatory_reporting}")

        lines.append("-}")
        return "\n".join(lines)

    def validate(self, mt: MT103) -> List[str]:
        """MT103 유효성 검증 → 오류 목록 반환"""

        errors = []

        # :20: Reference 검증
        if not mt.sender_ref:
            errors.append(":20: Reference 누락")
        elif len(mt.sender_ref) > 16:
            errors.append(f":20: Reference 16자 초과: {len(mt.sender_ref)}자")
        elif "//" in mt.sender_ref:
            errors.append(":20: Reference 이중슬래시(//) 금지")
        elif mt.sender_ref.startswith("/") or mt.sender_ref.endswith("/"):
            errors.append(":20: Reference 슬래시(/)로 시작/끝 금지")

        # :23B: 검증
        valid_op_codes = {"CRED", "CRTS", "SPAY", "SSTD"}
        if mt.bank_op_code not in valid_op_codes:
            errors.append(f":23B: 유효하지 않은 코드: {mt.bank_op_code}")

        # :32A: 검증
        if not mt.value_date:
            errors.append(":32A: Value Date 누락")
        if not mt.currency:
            errors.append(":32A: Currency 누락")
        elif len(mt.currency) != 3:
            errors.append(f":32A: Currency 3자리 아님: {mt.currency}")
        if not mt.amount or mt.amount <= 0:
            errors.append(":32A: Amount 누락 또는 0 이하")

        # :50: 송금의뢰인 검증
        if not mt.ordering_customer.name:
            errors.append(":50: 송금의뢰인 이름 누락")

        # :59: 수취인 검증
        if not mt.beneficiary.name:
            errors.append(":59: 수취인 이름 누락")
        if not mt.beneficiary.account:
            errors.append(":59: 수취인 계좌번호 누락")

        # :71A: 수수료 코드 검증
        if mt.charge_bearer not in {"BEN", "OUR", "SHA"}:
            errors.append(f":71A: 유효하지 않은 코드: {mt.charge_bearer}")

        # 역외원화 전용 검증
        if mt.currency == "KRW":
            if mt.amount and mt.amount != mt.amount.to_integral_value():
                errors.append(":32A: KRW는 소수점 불가")

        return errors

    # ── Private 메서드 ───────────────────────────────────

    def _extract_tags(self, raw: str) -> Dict[str, str]:
        """Block 4 태그 추출"""
        block4_match = self.BLOCK4_PATTERN.search(raw)
        content = block4_match.group(1) if block4_match else raw
        tags = {}
        for match in self.TAG_PATTERN.finditer(content):
            tags[match.group(1)] = match.group(2)
        return tags

    def _parse_32a(self, value: str) -> dict:
        match = self.AMOUNT_32A.match(value.strip())
        if not match:
            return {}
        ds = match.group("date")
        return {
            "date"     : date(int("20" + ds[:2]),
                              int(ds[2:4]),
                              int(ds[4:6])),
            "currency" : match.group("ccy"),
            "amount"   : Decimal(match.group("amount").replace(",", ".")),
        }

    def _parse_33b(self, value: str) -> dict:
        match = self.AMOUNT_33B.match(value.strip())
        if not match:
            return {}
        return {
            "currency" : match.group("ccy"),
            "amount"   : Decimal(match.group("amount").replace(",", ".")),
        }

    def _parse_customer(self, value: str, option: str) -> CustomerInfo:
        """고객 필드 파싱 (50K/50F/59/59F)"""
        lines   = value.strip().split("\n")
        info    = CustomerInfo()

        if not lines:
            return info

        # 계좌번호 (첫 줄 /로 시작)
        if lines[0].startswith("/"):
            info.account = lines[0][1:].strip()
            lines = lines[1:]

        if option == "F":
            # 구조화 형식 (1/이름, 2/주소, 3/국가/도시)
            for line in lines:
                if line.startswith("1/"):
                    info.name = line[2:].strip()
                elif line.startswith("2/"):
                    info.address.append(line[2:].strip())
                elif line.startswith("3/"):
                    parts = line[2:].split("/")
                    info.country = parts[0].strip() if parts else ""
                    info.city    = parts[1].strip() if len(parts) > 1 else ""
        else:
            # 자유텍스트 형식
            if lines:
                info.name    = lines[0].strip()
                info.address = [l.strip() for l in lines[1:] if l.strip()]

        return info


# ── 사용 예시 ─────────────────────────────────────────────

## 4. MT202 : Financial Institution Transfer

### 4-1. MT202이란?

```
MT202 = 금융기관간 자금이체 전문
      = 은행끼리 직접 자금 이동
      = MT103 처리 후 실제 자금 결제를 위한 전문

MT103 vs MT202 핵심 차이:
┌─────────────────┬──────────────────────────────────────────┐
│    MT103        │             MT202                        │
├─────────────────┼──────────────────────────────────────────┤
│ 고객 → 고객 송금 │ 은행 → 은행 자금이체                      │
│ 고객 정보 포함  │ 고객 정보 없음                            │
│ 지시 전문       │ 실제 자금 이동 전문                       │
│ 수취인 계좌     │ 수취 은행 계좌                            │
└─────────────────┴──────────────────────────────────────────┘

MT202 COV (Cover):
  MT103 Cover Payment 시 사용
  → 고객 정보가 :72: 또는 Sequence B에 포함
  → Block 3에 {103:COV} 태그 존재
```

### 4-2. MT202 전체 전문 예시

/ACC/OFFSHORE KRW SETTLEMENT
-}
{5:{CHK:BCDEF1234567}}
```

### 4-3. MT202 필드 완전 해설

#### :20: Sender's Reference

```
형식   : 16자 이내
필수   : M (Mandatory)
예시   : WOOREF240101001

MT103과 다른 점:
  → MT202 자체의 Reference
  → MT103 Reference와 다른 값 사용
  → 내부 추적용 별도 번호 채번

역외원화결제 설계:
  WOO + YYYYMMDD + 일련번호
  → WOOREF2024010100001
```

#### :21: Related Reference (관련 참조번호)

```
형식   : 16자 이내
필수   : M (Mandatory)
예시   : OWPREF240101001

★ MT202 핵심 필드
  → 이 MT202가 어떤 MT103과 연관되는지 연결
  → MT103의 :20: Reference 값 기재
  → 한국은행이 MT103↔MT202 매칭에 사용

역외원화결제:
  MT103 :20: = OWPREF240101001
  MT202 :21: = OWPREF240101001  ← 동일값!

특수값:
  NONREF = 특정 전문과 무관한 자금이체
  → 역외원화결제에서는 반드시 MT103 Reference 사용
```

#### :13C: Time Indication

```
형식   : /코드/HHMM+/-IINN
필수   : O (Optional)
예시   : /CLSTIME/1400+0900

MT103과 동일한 규칙 적용
한국은행 마감시간 기재 권장
```

#### :32A: Value Date / Currency / Amount

```
형식   : YYMMDD + CCY + Amount
필수   : M (Mandatory)
예시   : 240101KRW50000000,

MT103 :32A: 와 동일해야 함 ★
  → 날짜, 통화, 금액 모두 일치
  → 불일치 시 한국은행 처리 거절

역외원화결제 검증 포인트:
  ✅ MT103 :32A: = MT202 :32A: (필수 일치)
  ✅ 통화 = KRW
  ✅ 금액 = 원화 정수
```

#### :52A: / :52D: Ordering Institution (MT202 발신은행)

```
형식   : [/계좌] BIC
필수   : O (Optional)
예시   : :52A:WOOBURKRXXX

MT202에서의 의미:
  자금이체를 지시하는 은행 (우리은행)
  → 자금 출금 주체

역외원화결제:
  :52A:WOOBURKRXXX  ← 우리은행
```

#### :53A: / :53B: / :53D: Sender's Correspondent

```
형식   : [/계좌] BIC
필수   : O (Optional)
예시   : :53A:WOOBURKRXXX

MT202에서의 의미:
  발신은행(:52A:)의 환거래은행
  실제 자금 출금 계좌 보유 은행

역외원화결제:
  :53A:WOOBURKRXXX  ← 우리은행 (자기 자신)
  우리은행 계좌에서 자금 출금
```

#### :54A: / :54B: / :54D: Receiver's Correspondent

```
형식   : [/계좌] BIC
필수   : O (Optional)
예시   : :54A:HNBKKRSEXXX

MT202에서의 의미:
  수취은행(:58A:)의 환거래은행
  자금이 입금될 환거래 계좌 보유 은행

역외원화결제:
  :54A:HNBKKRSEXXX  ← 한국은행
```

#### :56A: / :56C: / :56D: Intermediary (중개기관)

```
형식   : [/계좌] BIC
필수   : O (Optional)
예시   : :56A:HVBKKRSEXXX

MT202에서의 의미:
  :57A: 또는 :58A: 에 도달하기 위한 중개 기관

역외원화결제:
  우리은행 ↔ 한국은행 직접 연결
  → 일반적으로 미사용
```

#### :57A: / :57B: / :57D: Account With Institution

```
형식   : [/계좌] BIC
필수   : O (Optional)
예시   : :57A:HNBKKRSE

MT202에서의 의미:
  수취인(:58A:) 계좌가 개설된 금융기관

역외원화결제:
  :57A:HNBKKRSE  ← 한국은행 ★
```

#### :58A: / :58D: Beneficiary Institution (수취 금융기관)

```
형식   : [/계좌] BIC  (:58A:)
         [/계좌] 이름  (:58D:)
필수   : M (Mandatory)
예시   : :58A:HNBKKRSE

★ MT202 핵심 필드 (MT103의 :59: 에 대응)
  → 최종 자금 수취 금융기관
  → 개인/법인 아닌 반드시 금융기관

MT103 vs MT202:
  MT103 :59: = 최종 수취인 (고객)
  MT202 :58: = 최종 수취 기관 (은행)

역외원화결제:
  :58A:HNBKKRSE  ← 한국은행 ★★★
  자금의 최종 목적지 = 한국은행
```

#### :72: Sender to Receiver Information

```
형식   : 최대 6줄 × 35자
필수   : O (Optional)
예시   : /INS/RELATED TO MT103 OWPREF240101001
         /ACC/OFFSHORE KRW SETTLEMENT

MT202 주요 사용 패턴:
  /INS/ = 관련 MT103 Reference 명시 (권장)
  /ACC/ = 계좌 관련 정보
  /BNF/ = 최종 수취인 정보 (COV 아닌 경우)
  /REJT/ = 거절 사유 (반환 시)
  /RETN/ = 반환 사유

역외원화결제:
  ✅ MT103 Reference 항상 명시
  ✅ 결제 목적 간략 기재
```

## 5. MT202 파싱 및 생성 코드

```python
from dataclasses import dataclass, field
from typing import Optional, List
from decimal import Decimal
from datetime import date
import re

@dataclass
class MT202:
    """MT202 도메인 객체"""

    # 필수 필드
    sender_ref      : str = ""
    related_ref     : str = ""              # :21: ★ MT103 연결
    value_date      : Optional[date] = None
    currency        : str = "KRW"
    amount          : Optional[Decimal] = None

    # 선택 필드
    time_indication : str = ""

    # 기관 정보
    ordering_inst        : str = ""         # :52A:
    senders_correspondent: str = ""         # :53A:
    receivers_corr       : str = ""         # :54A:
    intermediary         : str = ""         # :56A:
    account_with_inst    : str = ""         # :57A:
    beneficiary_inst     : str = ""         # :58A: ★ 한국은행

    # 부가정보
    sender_to_receiver   : str = ""         # :72:

    # COV 여부
    is_cover_payment     : bool = False


class MT202Parser:
    """
    MT202 전문 파서 및 생성기
    """

    TAG_PATTERN = re.compile(
        r':(\w{2,3}):(.*?)(?=\n:\w{2,3}:|\n-}|$)',
        re.DOTALL
    )
    BLOCK4_PATTERN = re.compile(
        r'\{4:\s*\n(.*?)\n-\}',
        re.DOTALL
    )
    AMOUNT_32A = re.compile(
        r'(?P<date>\d{6})(?P<ccy>[A-Z]{3})(?P<amount>[\d,]+)'
    )

    def parse(self, raw: str) -> MT202:
        """MT202 원문 파싱"""

        tags = self._extract_tags(raw)
        mt   = MT202()

        mt.sender_ref  = tags.get("20", "").strip()
        mt.related_ref = tags.get("21", "").strip()   # ★ MT103 Reference

        if "13C" in tags:
            mt.time_indication = tags["13C"].strip()

        if "32A" in tags:
            parsed = self._parse_32a(tags["32A"])
            mt.value_date = parsed["date"]
            mt.currency   = parsed["currency"]
            mt.amount     = parsed["amount"]

        mt.ordering_inst         = tags.get("52A", "").strip()
        mt.senders_correspondent = tags.get("53A", "").strip()
        mt.receivers_corr        = tags.get("54A", "").strip()
        mt.intermediary          = tags.get("56A", "").strip()
        mt.account_with_inst     = tags.get("57A", "").strip()
        mt.beneficiary_inst      = tags.get("58A", "").strip()
        mt.sender_to_receiver    = tags.get("72", "").strip()

        # COV 여부 확인 (Block 3)
        mt.is_cover_payment = "{103:COV}" in raw

        return mt

    def build(self, mt: MT202) -> str:
        """MT202 원문 생성"""

        lines = ["{4:"]

        lines.append(f":20:{mt.sender_ref}")
        lines.append(f":21:{mt.related_ref}")

        if mt.time_indication:
            lines.append(f":13C:{mt.time_indication}")

        date_str   = mt.value_date.strftime("%y%m%d")
        amount_str = str(mt.amount).replace(".", ",")
        lines.append(f":32A:{date_str}{mt.currency}{amount_str}")

        for tag, val in [
            ("52A", mt.ordering_inst),
            ("53A", mt.senders_correspondent),
            ("54A", mt.receivers_corr),
            ("56A", mt.intermediary),
            ("57A", mt.account_with_inst),
            ("58A", mt.beneficiary_inst),
        ]:
            if val:
                lines.append(f":{tag}:{val}")

        if mt.sender_to_receiver:
            lines.append(f":72:{mt.sender_to_receiver}")

        lines.append("-}")
        return "\n".join(lines)

    def validate(self, mt: MT202,
                 related_mt103: Optional[MT103] = None) -> List[str]:
        """MT202 유효성 검증"""

        errors = []

        if not mt.sender_ref:
            errors.append(":20: Reference 누락")

        if not mt.related_ref:
            errors.append(":21: Related Reference 누락 (MT103 연결 필수)")
        elif mt.related_ref == "NONREF":
            errors.append(":21: 역외원화결제는 NONREF 사용 불가")

        if not mt.amount or mt.amount <= 0:
            errors.append(":32A: 금액 누락 또는 0 이하")

        if not mt.beneficiary_inst:
            errors.append(":58A: 수취 금융기관 누락")

        # MT103 교차 검증
        if related_mt103:
            if mt.value_date != related_mt103.value_date:
                errors.append(
                    f":32A: Value Date MT103/MT202 불일치 "
                    f"({related_mt103.value_date} vs {mt.value_date})"
                )
            if mt.amount != related_mt103.amount:
                errors.append(
                    f":32A: Amount MT103/MT202 불일치 "
                    f"({related_mt103.amount} vs {mt.amount})"
                )
            if mt.related_ref != related_mt103.sender_ref:
                errors.append(
                    f":21: Related Ref MT103 Reference 불일치 "
                    f"({related_mt103.sender_ref} vs {mt.related_ref})"
                )

        return errors

    def _extract_tags(self, raw: str) -> dict:
        block4_match = self.BLOCK4_PATTERN.search(raw)
        content = block4_match.group(1) if block4_match else raw
        return {m.group(1): m.group(2)
                for m in self.TAG_PATTERN.finditer(content)}

    def _parse_32a(self, value: str) -> dict:
        match = self.AMOUNT_32A.match(value.strip())
        if not match:
            return {}
        ds = match.group("date")
        return {
            "date"     : date(int("20" + ds[:2]),
                              int(ds[2:4]),
                              int(ds[4:6])),
            "currency" : match.group("ccy"),
            "amount"   : Decimal(match.group("amount").replace(",", ".")),
        }


# ── 사용 예시 ─────────────────────────────────────────────

mt202_parser = MT202Parser()

raw_mt202 = """{1:F01WOOBURKRAXXX0000000002}
{2:I202HNBKKRSEXXX}
{4:
:20:WOOREF240101001
:21:OWPREF240101001
:32A:240101KRW50000000,
:52A:WOOBURKRXXX
:53A:WOOBURKRXXX
:54A:HNBKKRSEXXX
:57A:HNBKKRSE
:58A:HNBKKRSE
:72:/INS/RELATED TO MT103 OWPREF240101001
/ACC/OFFSHORE KRW SETTLEMENT
-}"""

mt202 = mt202_parser.parse(raw_mt202)

print("=" * 55)
print("MT202 파싱 결과")
print("=" * 55)
print(f"Reference      : {mt202.sender_ref}")
print(f"Related(MT103) : {mt202.related_ref}")   # ★ MT103 연결
print(f"통화/금액      : {mt202.currency} {mt202.amount:,}")
print(f"수취기관       : {mt202.beneficiary_inst}")  # HNBKKRSE
```

## 6. MT950 : Statement Message

### 6-1. MT950이란?

```
MT950 = 계좌 잔액 및 거래 명세 보고 전문
      = 한국은행이 우리은행에게 노스트로 계좌 현황 통보
      = 당일 결제 완료 확인 및 잔액 대사 용도

MT940 vs MT950:
┌──────────────┬────────────────────────────────────────────┐
│   MT940      │              MT950                         │
├──────────────┼────────────────────────────────────────────┤
│ 고객 계좌    │ 금융기관 계좌 (노스트로)                    │
│ 보고 전문    │ 보고 전문                                   │
│ 은행→고객   │ 은행→은행                                  │
│ 상세 거래    │ 잔액 중심 (간략한 거래)                     │
│ 명세 포함    │ 명세 포함                                   │
└──────────────┴────────────────────────────────────────────┘

역외원화결제에서 MT950 사용 목적:
  ✅ MT202 자금이체 후 한국은행 잔액 확인
  ✅ 일일 정산 마감 시 잔액 대사
  ✅ 노스트로 계좌 관리
```

### 6-3. MT950 필드 완전 해설

#### :20: Transaction Reference Number

```
형식   : 16자 이내
필수   : M (Mandatory)
예시   : BOKSTMT20240101001

한국은행이 발행하는 Statement Reference
역외원화결제:
  BOK + YYYYMMDD + 일련번호
  → BOKSTMT2024010100001
```

#### :25: Account Identification (계좌 식별)

```
형식   : 계좌번호 (최대 35자)
필수   : M (Mandatory)
예시   : KR12345678901234

★ 이 Statement가 어느 계좌에 대한 것인지
  우리은행의 한국은행 내 노스트로 계좌번호

형식 옵션:
  :25:  = 계좌번호만
  :25P: = 계좌번호 + BIC

:25P: 예시:
  KR12345678901234
  WOOBURKRXXX
```

#### :28C: Statement Number / Sequence Number

```
형식   : 5자리 / 5자리
필수   : M (Mandatory)
예시   : 00001/001

분해:
  00001 = Statement Number  (당해연도 누적 번호)
  001   = Sequence Number   (Statement 내 페이지)

복수 페이지:
  00001/001  = 1번 Statement 1페이지
  00001/002  = 1번 Statement 2페이지
  00001/003  = 1번 Statement 마지막 (LAST)

LAST 표시:
  00001/LAST  = 마지막 페이지 명시
```

#### :60F: / :60M: Opening Balance (기초 잔액)

```
형식   : D/C표시 + YYMMDD + CCY + Amount
필수   : M (Mandatory)
예시   : C240101KRW1000000000,

분해:
  C       = Credit (잔액이 예금 상태 = 플러스)
  D       = Debit  (잔액이 차입 상태 = 마이너스)
  240101  = 기준일 (2024-01-01)
  KRW     = 통화
  1000000000, = 금액 (10억원)

:60F: vs :60M:
  :60F: = Final Opening Balance   (최초 기초잔액)
  :60M: = Intermediate Opening Balance (중간 기초잔액, 복수 페이지)

역외원화결제:
  전일 마감 잔액 = 당일 :60F:
  ✅ 전일 :62F: = 당일 :60F: 일치 확인 필수
```

#### :61: Statement Line (거래 명세)

```
형식   : 복합 구조
필수   : O (Optional, 거래 없으면 생략)

┌──────────────────────────────────────────────────────────┐
│  :61: 필드 구조 완전 분해                                  │
├──────────────────────────────────────────────────────────┤
│                                                           │
│  :61:                                                     │
│  2401010101C50000000,NTRFOWPREF240101001                 │
│  │     │   │ │        │   │                              │
│  │     │   │ │        │   └── 참조번호 (Reference)        │
│  │     │   │ │        └────── 거래 유형 코드 (TRF, CHG..) │
│  │     │   │ └─────────────── N = 추가 정보 없음           │
│  │     │   └───────────────── C = Credit (입금)           │
│  │     │                      D = Debit  (출금)           │
│  │     └───────────────────── 역일 (Entry Date MMDD)      │
│  └─────────────────────────── 결제일 (Value Date YYMMDD)  │
│                                                           │
│  //WOOBURKRXXX                ← 추가 참조 (선택)          │
│  OFFSHORE KRW SETTLEMENT      ← 거래 설명 (선택)          │
└──────────────────────────────────────────────────────────┘

Value Date vs Entry Date:
  Value Date (240101) = 자금 효력 발생일 (6자리)
  Entry Date (0101)   = 계좌 기록일 (4자리 MMDD)
  → 통상 동일하나 다를 수 있음

D/C 표시:
  C = Credit  입금 (우리은행 입장에서 계좌 증가)
  D = Debit   출금 (우리은행 입장에서 계좌 감소)
  RD = Reversal Debit   (출금 취소)
  RC = Reversal Credit  (입금 취소)

거래유형 코드 (Transaction Type Code):
┌──────┬─────────────────────────────────────────────────┐
│ 코드 │ 설명                                             │
├──────┼─────────────────────────────────────────────────┤
│ NTRF │ Non-SWIFT Transfer (SWIFT 외 이체)               │
│ STRF │ SWIFT Transfer (SWIFT 이체) ← 역외원화 주로 사용 │
│ NCHG │ Non-SWIFT Charges (수수료)                       │
│ SCHG │ SWIFT Charges (SWIFT 수수료)                     │
│ NDIV │ Non-SWIFT Dividend (배당)                        │
│ NINT │ Non-SWIFT Interest (이자)                        │
│ NINP │ Non-SWIFT Interest Paid (이자 지급)              │
└──────┴─────────────────────────────────────────────────┘

:61: 완전 예시 (역외원화결제 MT202 관련):
  :61:2401010101C50000000,NTRFOWPREF240101001
  //WOOBURKRXXX
  OFFSHORE KRW SETTLEMENT

  → 2024-01-01 입금
  → 50,000,000 KRW Credit
  → Reference: OWPREF240101001 (MT103 Reference)
  → 추가: 우리은행에서 이체
```

#### :62F: / :62M: Closing Balance (기말 잔액)

```
형식   : D/C표시 + YYMMDD + CCY + Amount
필수   : M (Mandatory)
예시   : C240101KRW1047000000,

:62F: = Final Closing Balance     (최종 기말잔액)
:62M: = Intermediate Closing Bal. (중간 기말잔액)

역외원화결제:
  :60F: 기초잔액 + 당일 거래 합계 = :62F: 기말잔액
  검증 공식:
  :62F: = :60F: + ΣCredit - ΣDebit

당일 거래 후:
  기초: 1,000,000,000
  +입금: 50,000,000 (MT202 결제)
  -수수료: 3,000,000
  기말: 1,047,000,000
```

#### :64: Available Balance (가용 잔액)

```
형식   : D/C표시 + YYMMDD + CCY + Amount
필수   : O (Optional)
예시   : C240101KRW1047000000,

실제 인출/사용 가능한 잔액
→ 당일 마감 후 익영업일 사용 가능 잔액

:62F: vs :64:
  :62F: = 장부 기말잔액 (Book Balance)
  :64:  = 가용 잔액 (Available Balance)
  → 미결 거래, 홀드 금액 제외 후 실제 사용 가능 금액
```

#### :65: Forward Available Balance (선도 가용 잔액)

```
형식   : D/C표시 + YYMMDD + CCY + Amount
필수   : O (Optional, 복수 기재 가능)
예시   : C240102KRW1047000000,

미래 특정 시점의 예상 잔액
→ Value Date가 미래인 거래 포함 예상 잔액

복수 기재 예시:
  :65:C240102KRW1047000000,    ← D+1 예상잔액
  :65:C240103KRW1097000000,    ← D+2 예상잔액
```

#### :86: Information to Account Owner (계좌 소유자 정보)

```
형식   : 최대 6줄 × 65자
필수   : O (Optional, :61: 다음에 위치)
예시   : DAILY STATEMENT
         OFFSHORE KRW ACCOUNT
         REFERENCE DATE 20240101

사용 목적:
  :61: 거래에 대한 추가 설명
  또는 Statement 전체에 대한 설명

역외원화결제:
  ✅ 거래 목적, 관련 Reference 추가 기재
  ✅ 대사(Reconciliation) 시 활용
```

## 7. MT950 파싱 및 생성 코드

```python
from dataclasses import dataclass, field
from typing import Optional, List
from decimal import Decimal
from datetime import date
import re

@dataclass
class StatementLine:
    """MT950 :61: 거래 명세 라인"""
    value_date      : Optional[date] = None
    entry_date      : str = ""
    dc_mark         : str = ""         # C / D / RC / RD
    amount          : Optional[Decimal] = None
    transaction_code: str = ""         # NTRF, STRF, NCHG ...
    reference       : str = ""         # 거래 참조번호
    extra_ref       : str = ""         # // 추가 참조
    description     : str = ""         # 거래 설명
    account_info    : str = ""         # :86: 추가 정보

@dataclass
class BalanceField:
    """잔액 필드 공통 구조"""
    dc_mark  : str = ""              # C / D
    bal_date : Optional[date] = None
    currency : str = ""
    amount   : Optional[Decimal] = None

@dataclass
class MT950:
    """MT950 도메인 객체"""
    reference       : str = ""            # :20:
    account_id      : str = ""            # :25:
    statement_no    : str = ""            # :28C: Statement 번호
    sequence_no     : str = ""            # :28C: Sequence 번호
    opening_balance : BalanceField = field(
                          default_factory=BalanceField)  # :60F:
    statement_lines : List[StatementLine] = field(
                          default_factory=list)           # :61:
    closing_balance : BalanceField = field(
                          default_factory=BalanceField)  # :62F:
    available_bal   : Optional[BalanceField] = None      # :64:
    forward_bals    : List[BalanceField] = field(
                          default_factory=list)           # :65:


class MT950Parser:
    """
    MT950 전문 파서 및 생성기
    """

    TAG_PATTERN = re.compile(
        r':(\w{2,3}):(.*?)(?=\n:\w{2,3}:|\n-}|$)',
        re.DOTALL
    )
    BLOCK4_PATTERN = re.compile(
        r'\{4:\s*\n(.*?)\n-\}',
        re.DOTALL
    )

    # 잔액 필드 패턴 (60F/60M/62F/62M/64/65)
    BALANCE_PATTERN = re.compile(
        r'(?P<dc>[CD])'
        r'(?P<date>\d{6})'
        r'(?P<ccy>[A-Z]{3})'
        r'(?P<amount>[\d,]+)'
    )

    # :61: 거래 명세 패턴
    STMT_LINE_PATTERN = re.compile(
        r'(?P<vdate>\d{6})'          # Value Date
        r'(?P<edate>\d{4})?'         # Entry Date (선택)
        r'(?P<dc>R?[CD])'            # D/C Mark (RC/RD 포함)
        r'(?P<amount>[\d,]+)'        # Amount
        r',?'
        r'(?P<tcode>[A-Z]{4})'       # Transaction Code
        r'(?P<ref>.{0,16})'          # Reference
    )

    def parse(self, raw: str) -> MT950:
        """MT950 전문 파싱"""

        tags    = self._extract_tags(raw)
        mt      = MT950()
        lines   = list(tags.items())

        mt.reference = tags.get("20", "").strip()
        mt.account_id = tags.get("25", "").strip()

        # :28C: 파싱
        if "28C" in tags:
            parts = tags["28C"].strip().split("/")
            mt.statement_no = parts[0] if parts else ""
            mt.sequence_no  = parts[1] if len(parts) > 1 else ""

        # 잔액 필드 파싱
        for tag, attr in [
            ("60F", "opening_balance"),
            ("60M", "opening_balance"),
            ("62F", "closing_balance"),
            ("62M", "closing_balance"),
        ]:
            if tag in tags:
                setattr(mt, attr, self._parse_balance(tags[tag]))

        if "64" in tags:
            mt.available_bal = self._parse_balance(tags["64"])

        if "65" in tags:
            mt.forward_bals = [self._parse_balance(tags["65"])]

        # :61: 거래 명세 파싱
        mt.statement_lines = self._parse_statement_lines(
                                 tags.get("61", ""),
                                 tags.get("86", ""))

        return mt

    def validate_balance(self, mt: MT950) -> List[str]:
        """잔액 검증"""

        errors = []

        if not mt.opening_balance.amount:
            errors.append(":60F: 기초잔액 누락")
        if not mt.closing_balance.amount:
            errors.append(":62F: 기말잔액 누락")

        if (mt.opening_balance.amount and
            mt.closing_balance.amount and
            mt.statement_lines):

            # 기초 + Credit - Debit = 기말 검증
            total_credit = sum(
                line.amount for line in mt.statement_lines
                if line.dc_mark == "C" and line.amount
            )
            total_debit  = sum(
                line.amount for line in mt.statement_lines
                if line.dc_mark == "D" and line.amount
            )
            opening = mt.opening_balance.amount
            closing = mt.closing_balance.amount

            dc_multiplier = (1 if mt.opening_balance.dc_mark == "C"
                               else -1)
            calc_closing  = (opening * dc_multiplier
                             + total_credit - total_debit)

            if abs(calc_closing) != closing:
                errors.append(
                    f":62F: 잔액 불일치 "
                    f"(계산값: {calc_closing:,} "
                    f"/ 보고값: {closing:,})"
                )

        return errors

    def get_related_references(self, mt: MT950) -> List[str]:
        """MT950에서 MT103/MT202 Reference 추출"""

        refs = []
        for line in mt.statement_lines:
            if line.reference and line.reference != "NONREF":
                refs.append(line.reference)
        return refs

    def reconcile_with_mt202(self,
                              mt950    : MT950,
                              mt202_list: List[MT202]) -> dict:
        """MT950 거래명세와 MT202 대사"""

        result = {
            "matched"   : [],
            "unmatched_950" : [],
            "unmatched_202" : [],
        }

        mt202_refs = {mt.sender_ref: mt for mt in mt202_list}
        matched_refs = set()

        for line in mt950.statement_lines:
            if line.reference in mt202_refs:
                result["matched"].append({
                    "reference"    : line.reference,
                    "amount_950"   : line.amount,
                    "amount_202"   : mt202_refs[line.reference].amount,
                    "is_match"     : (line.amount ==
                                      mt202_refs[line.reference].amount),
                })
                matched_refs.add(line.reference)
            else:
                result["unmatched_950"].append(line.reference)

        for ref in mt202_refs:
            if ref not in matched_refs:
                result["unmatched_202"].append(ref)

        return result

    # ── Private 메서드 ───────────────────────────────────

    def _extract_tags(self, raw: str) -> dict:
        block4_match = self.BLOCK4_PATTERN.search(raw)
        content = block4_match.group(1) if block4_match else raw
        return {m.group(1): m.group(2)
                for m in self.TAG_PATTERN.finditer(content)}

    def _parse_balance(self, value: str) -> BalanceField:
        match = self.BALANCE_PATTERN.match(value.strip())
        if not match:
            return BalanceField()
        ds = match.group("date")
        return BalanceField(
            dc_mark  = match.group("dc"),
            bal_date = date(int("20" + ds[:2]),
                            int(ds[2:4]),
                            int(ds[4:6])),
            currency = match.group("ccy"),
            amount   = Decimal(match.group("amount").replace(",", ".")),
        )

    def _parse_statement_lines(self,
                                stmt_raw  : str,
                                info_raw  : str) -> List[StatementLine]:
        """
        :61: 거래 명세 복수 파싱
        :86: 추가 정보 연계
        """
        lines = []
        raw_entries = re.split(r'\n(?=\d{6})', stmt_raw.strip())

        for entry in raw_entries:
            if not entry.strip():
                continue

            entry_lines = entry.strip().split("\n")
            first_line  = entry_lines[0]
            match = self.STMT_LINE_PATTERN.match(first_line)

            if not match:
                continue

            vd = match.group("vdate")
            stmt_line = StatementLine(
                value_date       = date(int("20" + vd[:2]),
                                        int(vd[2:4]),
                                        int(vd[4:6])),
                entry_date       = match.group("edate") or "",
                dc_mark          = match.group("dc"),
                amount           = Decimal(
                                       match.group("amount")
                                       .replace(",", ".")),
                transaction_code = match.group("tcode"),
                reference        = match.group("ref").strip(),
            )

            # 추가 라인 처리
            for extra in entry_lines[1:]:
                if extra.startswith("//"):
                    stmt_line.extra_ref = extra[2:].strip()
                else:
                    stmt_line.description += extra.strip() + " "

            stmt_line.description = stmt_line.description.strip()
            lines.append(stmt_line)

        return lines


# ── 사용 예시 ─────────────────────────────────────────────

## 8. MT103 / MT202 / MT950 연계 검증

```python
class OWPMessageValidator:
    """
    역외원화결제 전문 연계 검증기
    MT103 → MT202 → MT950 일관성 검증
    """

    def validate_payment_chain(self,
                                mt103 : MT103,
                                mt202 : MT202,
                                mt950 : MT950) -> dict:
        """전문 체인 전체 검증"""

        result = {
            "mt103_mt202" : [],
            "mt202_mt950" : [],
            "overall"     : "PASS"
        }

        # ── MT103 ↔ MT202 검증 ──────────────────────────
        # 금액 일치
        if mt103.amount != mt202.amount:
            result["mt103_mt202"].append(
                f"금액 불일치: MT103={mt103.amount:,} "
                f"/ MT202={mt202.amount:,}"
            )

        # 통화 일치
        if mt103.currency != mt202.currency:
            result["mt103_mt202"].append(
                f"통화 불일치: MT103={mt103.currency} "
                f"/ MT202={mt202.currency}"
            )

        # Value Date 일치
        if mt103.value_date != mt202.value_date:
            result["mt103_mt202"].append(
                f"Value Date 불일치: MT103={mt103.value_date} "
                f"/ MT202={mt202.value_date}"
            )

        # MT202 :21: = MT103 :20:
        if mt202.related_ref != mt103.sender_ref:
            result["mt103_mt202"].append(
                f"Reference 연결 오류: "
                f"MT103:20:={mt103.sender_ref} "
                f"/ MT202:21:={mt202.related_ref}"
            )

        # 수취기관 검증 (한국은행)
        if "HNBK" not in mt202.beneficiary_inst:
            result["mt103_mt202"].append(
                f"수취기관 오류: {mt202.beneficiary_inst} "
                f"(한국은행 아님)"
            )

        # ── MT202 ↔ MT950 검증 ──────────────────────────
        # MT950 거래명세에서 MT202 Reference 확인
        stmt_refs = [
            line.reference
            for line in mt950.statement_lines
        ]

        if mt202.sender_ref not in stmt_refs:
            result["mt202_mt950"].append(
                f"MT202 Reference MT950 미확인: {mt202.sender_ref}"
            )

        # 입금 금액 일치
        credit_lines = [
            line for line in mt950.statement_lines
            if line.dc_mark == "C"
            and line.reference == mt202.sender_ref
        ]

        if credit_lines:
            stmt_amount = credit_lines[0].amount
            if stmt_amount != mt202.amount:
                result["mt202_mt950"].append(
                    f"입금금액 불일치: MT202={mt202.amount:,} "
                    f"/ MT950={stmt_amount:,}"
                )

        # 전체 결과
        all_errors = (result["mt103_mt202"] +
                      result["mt202_mt950"])
        result["overall"] = "FAIL" if all_errors else "PASS"

        return result


# ── 연계 검증 사용 예시 ───────────────────────────────────

validator = OWPMessageValidator()

chain_result = validator.validate_payment_chain(
    mt103  = mt103,
    mt202  = mt202,
    mt950  = mt950
)

print("\n" + "=" * 55)
print("전문 연계 검증 결과")
print("=" * 55)
print(f"전체결과       : {chain_result['overall']}")

if chain_result["mt103_mt202"]:
    print("\nMT103↔MT202 오류:")
    for e in chain_result["mt103_mt202"]:
        print(f"  ❌ {e}")

if chain_result["mt202_mt950"]:
    print("\nMT202↔MT950 오류:")
    for e in chain_result["mt202_mt950"]:
        print(f"  ❌ {e}")

if chain_result["overall"] == "PASS":
    print("\n✅ 모든 전문 연계 검증 통과")
