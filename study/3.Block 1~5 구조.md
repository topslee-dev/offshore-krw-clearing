## 1. SWIFT 전문 전체 구조 한눈에 보기

```
┌─────────────────────────────────────────────────────────────────────┐

│                                                                      │
│  {1:F01WOOBURKRAXXX0000000000}        ← Block 1 : Basic Header      │
│  {2:I103HNBKKRSEXXX}                  ← Block 2 : App Header        │
│  {3:{108:OWP20240101001}}             ← Block 3 : User Header       │
│  {4:                                  ← Block 4 : Text (본문) ★     │
│  :20:OWPREF240101001                                                 │
│  :23B:CRED                                                           │
│  :32A:240101KRW50000000,                                             │
│  :50K:/12345678901234                                                │
│  OVERSEAS COMPANY NEW YORK                                           │
│  :57A:HNBKKRSE                                                       │
│  :59:/98765432109876                                                 │
│  KOREAN IMPORT COMPANY                                               │
│  :71A:SHA                                                            │
│  -}                                   ← Block 4 종료                 │
│  {5:{CHK:ABCDEF123456}}              ← Block 5 : Trailer            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Block 역할 요약

```
┌─────────┬──────────────────┬────────────────────────────────────────┐
│  Block  │      이름         │               역할                      │
├─────────┼──────────────────┼────────────────────────────────────────┤
│ Block 1 │ Basic Header     │ 전문을 보내는 기관 식별 (발신자)          │
│ Block 2 │ App Header       │ 전문 종류 + 수신기관 식별                │
│ Block 3 │ User Header      │ 추가 옵션 정보 (Reference 등)            │
│ Block 4 │ Text Block       │ 실제 결제/금융 데이터 ★ 핵심             │
│ Block 5 │ Trailer          │ 무결성 검증 (체크섬, 암호화 등)           │
└─────────┴──────────────────┴────────────────────────────────────────┘
```

## 2. Block 1 : Basic Header Block

### 구조 분해

```
{1:F01WOOBURKRAXXX0000000000}
  │ │  │           │   │
  │ │  │           │   └── Sequence Number  (6자리)
  │ │  │           └─────── Session Number  (4자리)
  │ │  └─────────────────── BIC + LT Code  (12자리)
  │ └────────────────────── Service ID      (2자리)
  └──────────────────────── Application ID  (1자리)
```

### 각 필드 상세

```
┌──────────────────────────────────────────────────────────────────┐
│  Block 1 필드 상세                                                 │
├──────────────┬──────────┬──────────────────────────────────────┤
│  필드명       │  값      │  설명                                  │
├──────────────┼──────────┼──────────────────────────────────────┤
│ Application  │ F        │ F = FIN (금융 전문)                    │
│ ID           │          │ A = GPA (일반 애플리케이션)             │
│              │          │ L = Logins/System Messages            │
├──────────────┼──────────┼──────────────────────────────────────┤
│ Service ID   │ 01       │ 01 = FIN 서비스 (가장 일반적)           │
│              │          │ 21 = ACK/NAK 응답                     │
├──────────────┼──────────┼──────────────────────────────────────┤
│ LT Address   │ WOOB     │ BIC 앞 4자리 (은행코드)                │
│ (BIC+LT)     │ UR       │ 국가코드                               │
│ 12자리        │ KR       │ 지역코드                               │
│              │ A        │ LT Code (Logical Terminal A~Z)        │
│              │ XXX      │ Branch Code                           │
├──────────────┼──────────┼──────────────────────────────────────┤
│ Session      │ 0000     │ SWIFT 세션 번호 (4자리)                 │
│ Number       │          │ 연결 세션 식별                          │
├──────────────┼──────────┼──────────────────────────────────────┤
│ Sequence     │ 000000   │ 전문 순서 번호 (6자리)                  │
│ Number       │          │ 세션 내 전문 순서                       │
└──────────────┴──────────┴──────────────────────────────────────┘
```

### LT Code (Logical Terminal) 상세

```
LT Code = 동일 BIC로 여러 SWIFT 연결을 구분하는 코드
          A ~ Z 중 1자리 사용

WOOBURKR A XXX  → 우리은행 메인 채널 (LT-A)
WOOBURKR B XXX  → 우리은행 백업 채널 (LT-B)
HNBKKRSE A XXX  → 한국은행 메인 채널 (LT-A)

실무 포인트:
  ✅ 대부분 A를 메인으로 사용
  ✅ 고가용성을 위해 B 채널 구성
  ✅ Block 1에서 LT Code 누락 시 전문 오류
```

### Block 1 파싱 코드

```python
import re
from dataclasses import dataclass

@dataclass
class Block1:
    application_id  : str   # F, A, L
    service_id      : str   # 01, 21
    lt_address      : str   # WOOBURKRAXXX (12자리)
    bank_code       : str   # WOOB
    country_code    : str   # KR
    location_code   : str   # KR
    lt_code         : str   # A
    branch_code     : str   # XXX
    session_number  : str   # 0000
    sequence_number : str   # 000000

class Block1Parser:
    """
    SWIFT Block 1 (Basic Header) 파서
    형식: {1:AppID + ServiceID + LTAddress(12) + Session(4) + Seq(6)}
    """
    
    PATTERN = re.compile(
        r'\{1:'
        r'(?P<app_id>[FAL])'           # Application ID (1자리)
        r'(?P<svc_id>\d{2})'           # Service ID (2자리)
        r'(?P<bank_code>[A-Z]{4})'     # 은행코드 (4자리)
        r'(?P<country>[A-Z]{2})'       # 국가코드 (2자리)
        r'(?P<location>[A-Z0-9]{2})'   # 지역코드 (2자리)
        r'(?P<lt_code>[A-Z])'          # LT Code (1자리)
        r'(?P<branch>[A-Z0-9]{3})'     # Branch (3자리)
        r'(?P<session>\d{4})'          # Session Number (4자리)
        r'(?P<sequence>\d{6})'         # Sequence Number (6자리)
        r'\}'
    )
    
    def parse(self, raw: str) -> Block1:
        match = self.PATTERN.search(raw)
        
        if not match:
            raise ValueError(f"Block 1 파싱 오류: {raw}")
        
        return Block1(
            application_id  = match.group("app_id"),
            service_id      = match.group("svc_id"),
            lt_address      = (match.group("bank_code") +
                               match.group("country") +
                               match.group("location") +
                               match.group("lt_code") +
                               match.group("branch")),
            bank_code       = match.group("bank_code"),
            country_code    = match.group("country"),
            location_code   = match.group("location"),
            lt_code         = match.group("lt_code"),
            branch_code     = match.group("branch"),
            session_number  = match.group("session"),
            sequence_number = match.group("sequence"),
        )
    
    def build(self, 
              sender_bic  : str,
              lt_code     : str = "A",
              session     : str = "0000",
              sequence    : str = "000000",
              app_id      : str = "F",
              service_id  : str = "01") -> str:
        """Block 1 생성"""
        
        # BIC 정규화 (8자리)
        bic_8 = sender_bic.upper()[:8]
        
        return (f"{{1:{app_id}{service_id}"
                f"{bic_8[:4]}"          # 은행코드
                f"{bic_8[4:6]}"         # 국가코드
                f"{bic_8[6:8]}"         # 지역코드
                f"{lt_code}"            # LT Code
                f"XXX"                  # Branch
                f"{session}"            # Session
                f"{sequence}}}")        # Sequence


# ── 사용 예시 ─────────────────────────────────────────────

parser = Block1Parser()

raw = "{1:F01WOOBURKRAXXX0000000000}"
b1  = parser.parse(raw)

print(f"Application ID  : {b1.application_id}")   # F
print(f"Service ID      : {b1.service_id}")        # 01
print(f"LT Address      : {b1.lt_address}")        # WOOBURKRAXXX
print(f"은행코드         : {b1.bank_code}")          # WOOB
print(f"국가코드         : {b1.country_code}")       # KR (위치상 UR)
print(f"LT Code         : {b1.lt_code}")            # A
print(f"Sequence        : {b1.sequence_number}")    # 000000

# Block 1 생성
built = parser.build(sender_bic="WOOBURKR", lt_code="A")
print(built)
# {1:F01WOOBURKRAXXX0000000000}
```

## 3. Block 2 : Application Header Block

### Block 2의 두 가지 형태

```
Block 2는 입력(Input)과 출력(Output) 두 가지 형태가 있음

Input  (I) = 우리은행이 다른 기관으로 전문 발송할 때
Output (O) = 우리은행이 다른 기관으로부터 전문 수신할 때
```

### Block 2 Input (발신 전문)

```
{2:I103HNBKKRSEXXX}
   │ │  │
   │ │  └──── 수신기관 BIC (15자리 = BIC12 + Priority)
   │ └──────── MT Type (3자리)
   └────────── I = Input (발신)

상세 분해:
{2:I 103 HNBKKRSE XXX N}
      │   │        │   │
      │   │        │   └── Priority (N=Normal, U=Urgent, S=System)
      │   │        └─────── Branch Code
      │   └──────────────── 수신기관 BIC 8자리
      └──────────────────── MT Type
```

### Block 2 Output (수신 전문)

```
{2:O103 1430 240101HNBKKRSEAXXX00001234561 240101 1435 N}
   │ │   │    │                            │       │    │
   │ │   │    │                            │       │    └─ Priority
   │ │   │    │                            │       └─────── 수신시각
   │ │   │    │                            └─────────────── 수신날짜
   │ │   │    └──────────────────────────── 발신 전문 참조
   │ │   └───────────────────────────────── 발신시각
   │ └───────────────────────────────────── MT Type
   └───────────────────────────────────────  O = Output (수신)
```

### Block 2 필드 상세

```
┌──────────────────────────────────────────────────────────────────┐
│  Block 2 Input 필드                                               │
├──────────────────┬───────────────────────────────────────────────┤
│  필드             │  설명                                          │
├──────────────────┼───────────────────────────────────────────────┤
│ I/O 구분          │ I = 발신 / O = 수신                           │
│ MT Type          │ 103, 202, 950 등 메시지 유형 번호              │
│ Receiver BIC     │ 수신기관 BIC (12자리 = BIC8 + LT + Branch)    │
│ Priority         │ N=Normal, U=Urgent, S=System                  │
└──────────────────┴───────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────┐
│  Block 2 Output 추가 필드                                          │
├──────────────────┬───────────────────────────────────────────────┤
│  필드             │  설명                                          │
├──────────────────┼───────────────────────────────────────────────┤
│ Input Time       │ 발신측이 전문 보낸 시각 (HHMM)                  │
│ MIR              │ Message Input Reference                        │
│                  │ = 날짜 + 발신BIC + Session + Sequence          │
│ Output Date      │ SWIFT가 수신측에 전달한 날짜                    │
│ Output Time      │ SWIFT가 수신측에 전달한 시각                    │
└──────────────────┴───────────────────────────────────────────────┘
```

### Priority 코드

```
Priority 코드 상세

N = Normal   (일반)
    → 일반 결제 전문
    → 역외원화결제 일반 건

U = Urgent   (긴급)
    → 당일 마감 임박 전문
    → 고액 긴급 결제

S = System   (시스템)
    → SWIFT 시스템 전문
    → 일반 업무 전문에는 미사용
```

### MT Type 번호 체계

```
MT Type 번호 = 3자리 숫자

첫 번째 자리 = 카테고리 (Category)

  1xx → Customer Payments & Checks (고객 결제)
        MT103 = 고객 송금 ★
        MT110 = 수표
        MT111 = 수표 취소

  2xx → Financial Institution Transfers (금융기관 이체)
        MT200 = 자기앞 이체
        MT202 = 금융기관 이체 ★
        MT205 = 3자 이체

  3xx → Treasury Markets (외환/파생)
        MT300 = FX 확인
        MT320 = FX 정산

  4xx → Collections & Cash Letters (추심)
        MT400 = 추심 결제

  5xx → Securities Markets (증권)
        MT515 = 증권 거래 확인

  6xx → Precious Metals (귀금속)

  7xx → Documentary Credits (신용장)
        MT700 = 신용장 개설
        MT740 = 상환 수권

  8xx → Travellers Cheques (여행자수표)

  9xx → Cash Management (현금 관리)
        MT900 = 출금 확인 ★
        MT910 = 입금 확인 ★
        MT940 = 고객 계좌 보고
        MT950 = 잔액 보고 ★
        MT999 = 자유형식 전문
```

### Block 2 파싱 코드

```python
import re
from dataclasses import dataclass
from typing import Optional

@dataclass
class Block2Input:
    direction       : str = "I"
    mt_type         : str = ""
    receiver_bic    : str = ""
    priority        : str = "N"

@dataclass
class Block2Output:
    direction       : str = "O"
    mt_type         : str = ""
    input_time      : str = ""
    mir             : str = ""   # Message Input Reference
    output_date     : str = ""
    output_time     : str = ""
    priority        : str = "N"

class Block2Parser:
    """
    SWIFT Block 2 (Application Header) 파서
    Input / Output 두 형태 지원
    """
    
    # Input 패턴
    INPUT_PATTERN = re.compile(
        r'\{2:'
        r'(?P<direction>I)'
        r'(?P<mt_type>\d{3})'
        r'(?P<receiver_bic>[A-Z0-9]{12,15})'
        r'(?P<priority>[NUS])?'
        r'\}'
    )
    
    # Output 패턴
    OUTPUT_PATTERN = re.compile(
        r'\{2:'
        r'(?P<direction>O)'
        r'(?P<mt_type>\d{3})'
        r'(?P<input_time>\d{4})'
        r'(?P<mir>\S+)'
        r'\s+'
        r'(?P<output_date>\d{6})'
        r'(?P<output_time>\d{4})'
        r'(?P<priority>[NUS])?'
        r'\}'
    )
    
    def parse(self, raw: str) -> Block2Input | Block2Output:
        """Block 2 파싱 (Input/Output 자동 감지)"""
        
        if "{2:I" in raw:
            return self._parse_input(raw)
        elif "{2:O" in raw:
            return self._parse_output(raw)
        else:
            raise ValueError(f"Block 2 형식 오류: {raw}")
    
    def _parse_input(self, raw: str) -> Block2Input:
        match = self.INPUT_PATTERN.search(raw)
        if not match:
            raise ValueError(f"Block 2 Input 파싱 오류: {raw}")
        
        return Block2Input(
            direction    = "I",
            mt_type      = match.group("mt_type"),
            receiver_bic = match.group("receiver_bic"),
            priority     = match.group("priority") or "N",
        )
    
    def _parse_output(self, raw: str) -> Block2Output:
        match = self.OUTPUT_PATTERN.search(raw)
        if not match:
            raise ValueError(f"Block 2 Output 파싱 오류: {raw}")
        
        return Block2Output(
            direction   = "O",
            mt_type     = match.group("mt_type"),
            input_time  = match.group("input_time"),
            mir         = match.group("mir"),
            output_date = match.group("output_date"),
            output_time = match.group("output_time"),
            priority    = match.group("priority") or "N",
        )
    
    def build_input(self,
                    mt_type      : str,
                    receiver_bic : str,
                    priority     : str = "N") -> str:
        """Block 2 Input 생성"""
        
        bic = receiver_bic.upper()
        
        # BIC 12자리 정규화 (BIC8 + LT + Branch)
        if len(bic) == 8:
            bic = bic[:6] + "X" + bic[6:] + "XX"  # LT=X, Branch=XX
        
        return f"{{2:I{mt_type}{bic}{priority}}}"
    
    def get_mt_description(self, mt_type: str) -> str:
        """MT Type 설명 반환"""
        descriptions = {
            "103" : "고객 송금 (Customer Credit Transfer)",
            "200" : "자기앞 자금이체 (Financial Institution Transfer)",
            "202" : "금융기관 자금이체 (FI Transfer for Own Account)",
            "205" : "3자 금융기관 이체",
            "900" : "출금 확인 (Confirmation of Debit)",
            "910" : "입금 확인 (Confirmation of Credit)",
            "940" : "고객 계좌 보고 (Customer Statement)",
            "950" : "잔액 보고 (Statement Message)",
            "195" : "조회 전문 (Queries)",
            "196" : "응신 전문 (Answers)",
            "199" : "자유형식 조회 (Free Format Message)",
        }
        return descriptions.get(mt_type, f"알 수 없는 MT Type: {mt_type}")


# ── 사용 예시 ─────────────────────────────────────────────

parser = Block2Parser()

# Input 파싱
raw_input  = "{2:I103HNBKKRSEXXX}"
b2_input   = parser.parse(raw_input)
print(f"MT Type      : {b2_input.mt_type}")        # 103
print(f"수신 BIC     : {b2_input.receiver_bic}")   # HNBKKRSEXXX
print(f"설명         : {parser.get_mt_description(b2_input.mt_type)}")

# Input Block 2 생성
built = parser.build_input(
    mt_type      = "103",
    receiver_bic = "HNBKKRSE",
    priority     = "N"
)
print(built)
# {2:I103HNBKKRSEXXX N}
```

## 4. Block 3 : User Header Block

### 구조

```
{3:{108:OWP20240101001}{119:STP}{121:a1b2c3d4-e5f6-7890-abcd-ef1234}}
    │                   │        │
    │                   │        └── {121:} UETR (MX 호환용)
    │                   └─────────── {119:} Validation Flag
    └─────────────────────────────── {108:} MUR (Message User Reference)
```

### Block 3 주요 서브필드 (Tag)

```
┌──────────────────────────────────────────────────────────────────────┐
│  Block 3 주요 태그 목록                                                │
├────────┬──────────────────────────┬───────────────────────────────────┤
│  태그  │  이름                     │  설명                              │
├────────┼──────────────────────────┼───────────────────────────────────┤
│ {103:} │ Service Identifier       │ COV = Cover Payment               │
├────────┼──────────────────────────┼───────────────────────────────────┤
│ {106:} │ Message Input Reference  │ 원본 전문 참조 (MIR)               │
├────────┼──────────────────────────┼───────────────────────────────────┤
│ {108:} │ Message User Reference   │ 사용자 정의 Reference              │
│        │ (MUR)                    │ ★ 내부 시스템 추적용               │
├────────┼──────────────────────────┼───────────────────────────────────┤
│ {111:} │ Service Type Identifier  │ 001 = Priority / gpi              │
├────────┼──────────────────────────┼───────────────────────────────────┤
│ {121:} │ UETR                     │ gpi 거래 추적 ID (UUID v4)         │
│        │                          │ ★ SWIFT gpi 필수 필드              │
├────────┼──────────────────────────┼───────────────────────────────────┤
│ {119:} │ Validation Flag          │ STP = Straight Through Processing │
│        │                          │ REMIT = Remittance Info 포함       │
│        │                          │ COV = Cover Payment                │
└────────┴──────────────────────────┴───────────────────────────────────┘
```

### STP (Straight Through Processing) 이해

```
STP = 자동 처리 (수동 개입 없이)

{119:STP} 태그가 있으면:
  → 모든 필드가 표준 형식에 맞아야 함
  → 자동 처리 가능 전문으로 표시
  → 처리 속도 향상 / 수수료 절감 가능
  → 한 글자라도 형식 오류 시 STP 실패 → 수동 처리

{119:STP} 없으면:
  → 담당자가 수동 확인 후 처리
  → 처리 지연 가능성
  → 추가 수수료 발생 가능

역외원화결제 실무 포인트:
  ✅ STP 처리 목표로 전문 품질 관리 필요
  ✅ :50K: 주소 형식 오류가 STP 실패 주요 원인
  ✅ 한국은행 연동 전문은 STP 권장
```

### UETR (gpi) 처리

```
SWIFT gpi (global payments innovation) 연동 시

{3:
  {111:001}                                    ← gpi 서비스 타입
  {121:a1b2c3d4-e5f6-7890-abcd-ef1234567890}  ← UETR (UUID v4)
}

UETR 규칙:
  ✅ 최초 발신 은행이 생성
  ✅ 전 구간 동일 UETR 유지 (변경 금지)
  ✅ MT 전문에서는 Block 3에 위치
  ✅ MX 전문에서는 <UETR> 태그로 위치
```

### Block 3 파싱 코드

```python
import re
import uuid
from dataclasses import dataclass, field
from typing import Optional

@dataclass
class Block3:
    service_identifier  : Optional[str] = None  # {103:}
    mir                 : Optional[str] = None  # {106:}
    mur                 : Optional[str] = None  # {108:} ★
    service_type        : Optional[str] = None  # {111:}
    uetr                : Optional[str] = None  # {121:} ★
    validation_flag     : Optional[str] = None  # {119:}
    raw_tags            : dict = field(default_factory=dict)

class Block3Parser:
    """
    SWIFT Block 3 (User Header) 파서
    중괄호 기반 서브태그 파싱
    """
    
    # 서브태그 패턴: {태그번호:값}
    SUBTAG_PATTERN = re.compile(r'\{(\d{3}):([^}]*)\}')
    
    def parse(self, raw: str) -> Block3:
        """Block 3 전체 파싱"""
        
        # {3: ... } 추출
        block3_match = re.search(r'\{3:(.*?)\}(?=\{[12456])', raw, re.DOTALL)
        if not block3_match:
            return Block3()  # Block 3 없음 (선택 필드)
        
        block3_content = block3_match.group(1)
        raw_tags = {}
        
        for match in self.SUBTAG_PATTERN.finditer(block3_content):
            tag_num = match.group(1)
            value   = match.group(2)
            raw_tags[tag_num] = value
        
        return Block3(
            service_identifier = raw_tags.get("103"),
            mir                = raw_tags.get("106"),
            mur                = raw_tags.get("108"),
            service_type       = raw_tags.get("111"),
            uetr               = raw_tags.get("121"),
            validation_flag    = raw_tags.get("119"),
            raw_tags           = raw_tags,
        )
    
    def build(self,
              mur             : Optional[str] = None,
              uetr            : Optional[str] = None,
              validation_flag : Optional[str] = None,
              service_type    : Optional[str] = None,
              is_gpi          : bool = False) -> str:
        """Block 3 생성"""
        
        tags = []
        
        if mur:
            tags.append(f"{{108:{mur}}}")
        
        if is_gpi or uetr:
            tags.append("{111:001}")
            _uetr = uetr or str(uuid.uuid4())
            tags.append(f"{{121:{_uetr}}}")
        
        if validation_flag:
            tags.append(f"{{119:{validation_flag}}}")
        
        if not tags:
            return ""
        
        return "{3:" + "".join(tags) + "}"
    
    def is_stp(self, block3: Block3) -> bool:
        """STP 전문 여부"""
        return block3.validation_flag == "STP"
    
    def is_gpi(self, block3: Block3) -> bool:
        """gpi 전문 여부 (UETR 존재)"""
        return block3.uetr is not None
    
    def is_cover_payment(self, block3: Block3) -> bool:
        """Cover Payment 여부"""
        return (block3.service_identifier == "COV" or
                block3.validation_flag == "COV")


# ── 사용 예시 ─────────────────────────────────────────────

parser = Block3Parser()

raw = """{1:F01WOOBURKRAXXX0000000000}
{2:I103HNBKKRSEXXX}
{3:{108:OWP20240101001}{119:STP}{121:a1b2c3d4-e5f6-7890-abcd-ef1234567890}}
{4:
:20:OWPREF240101001
-}"""

b3 = parser.parse(raw)
print(f"MUR           : {b3.mur}")              # OWP20240101001
print(f"Validation    : {b3.validation_flag}")  # STP
print(f"UETR          : {b3.uetr}")             # a1b2c3d4-...
print(f"STP 여부      : {parser.is_stp(b3)}")   # True
print(f"gpi 여부      : {parser.is_gpi(b3)}")   # True

# Block 3 생성 (gpi + STP)
built = parser.build(
    mur             = "OWP20240101001",
    validation_flag = "STP",
    is_gpi          = True
)
print(built)
# {3:{108:OWP20240101001}{111:001}{121:UUID...}{119:STP}}
```

### 태그 옵션 코드 이해

```
태그명 규칙: :숫자[옵션코드]:

:50K:  → 태그 50, 옵션 K
:50F:  → 태그 50, 옵션 F (다른 형식)
:50A:  → 태그 50, 옵션 A (또 다른 형식)

옵션 코드별 의미:
  A = BIC 방식 (BIC Code로 기관 식별)
  D = 이름+주소 방식 (자유텍스트)
  F = 구조화 방식 (코드+값 분리)
  K = 자유텍스트 방식 (가장 일반적)

예시:
  :57A:HNBKKRSE         → BIC로 은행 지정
  :57D:/KR123456         → 계좌번호+이름으로 지정
         BANK OF KOREA
```

### Block 4 파싱 코드

```python
import re
from dataclasses import dataclass, field
from typing import Optional, Dict
from decimal import Decimal
from datetime import date

@dataclass
class MT103Data:
    """MT103 파싱 결과 도메인 객체"""
    
    # 필수 필드
    sender_reference    : str = ""           # :20:
    bank_op_code        : str = ""           # :23B:
    value_date          : Optional[date] = None  # :32A: 날짜
    currency            : str = ""           # :32A: 통화
    amount              : Optional[Decimal] = None  # :32A: 금액
    
    # 당사자 정보
    ordering_customer   : dict = field(default_factory=dict)  # :50K/F/A:
    ordering_institution: str = ""           # :52A:
    sender_correspondent: str = ""           # :53A:
    receiver_corr       : str = ""           # :54A:
    intermediary        : str = ""           # :56A:
    account_with_inst   : str = ""           # :57A: ★ 한국은행
    beneficiary         : dict = field(default_factory=dict)  # :59:
    
    # 부가 정보
    remittance_info     : str = ""           # :70:
    charge_bearer       : str = ""           # :71A:
    sender_to_receiver  : str = ""           # :72:
    regulatory_reporting: str = ""           # :77B:
    
    # 원본 태그 전체
    raw_tags            : Dict[str, str] = field(default_factory=dict)


class Block4Parser:
    """
    SWIFT Block 4 (Text Block) 파서
    MT103 기준 구현
    """
    
    # Block 4 추출 패턴
    BLOCK4_PATTERN = re.compile(
        r'\{4:\s*\n?(.*?)\n?-\}',
        re.DOTALL
    )
    
    # 태그 분리 패턴
    TAG_PATTERN = re.compile(
        r'(?:^|\n):(\w{1,3}):(.+?)(?=\n:\w{1,3}:|$)',
        re.DOTALL
    )
    
    def parse_block4(self, raw_message: str) -> Dict[str, str]:
        """Block 4에서 태그 딕셔너리 추출"""
        
        block4_match = self.BLOCK4_PATTERN.search(raw_message)
        if not block4_match:
            raise ValueError("Block 4를 찾을 수 없습니다")
        
        block4_content = block4_match.group(1)
        tags = {}
        
        for match in self.TAG_PATTERN.finditer(block4_content):
            tag   = match.group(1).strip()
            value = match.group(2).strip()
            tags[tag] = value
        
        return tags
    
    def parse_mt103(self, raw_message: str) -> MT103Data:
        """MT103 완전 파싱"""
        
        tags = self.parse_block4(raw_message)
        
        # :32A: 파싱
        amount_data = self._parse_32a(tags.get("32A", ""))
        
        # :50K/F/A: 파싱
        ordering = self._parse_customer_field(
            tags.get("50K") or tags.get("50F") or tags.get("50A", "")
        )
        
        # :59:/F: 파싱
        beneficiary = self._parse_customer_field(
            tags.get("59") or tags.get("59F") or tags.get("59A", "")
        )
        
        return MT103Data(
            sender_reference     = tags.get("20", ""),
            bank_op_code         = tags.get("23B", ""),
            value_date           = amount_data.get("value_date"),
            currency             = amount_data.get("currency", ""),
            amount               = amount_data.get("amount"),
            ordering_customer    = ordering,
            ordering_institution = tags.get("52A", ""),
            sender_correspondent = tags.get("53A", ""),
            receiver_corr        = tags.get("54A", ""),
            intermediary         = tags.get("56A", ""),
            account_with_inst    = tags.get("57A", ""),
            beneficiary          = beneficiary,
            remittance_info      = tags.get("70", ""),
            charge_bearer        = tags.get("71A", ""),
            sender_to_receiver   = tags.get("72", ""),
            regulatory_reporting = tags.get("77B", ""),
            raw_tags             = tags,
        )
    
    def _parse_32a(self, value: str) -> dict:
        """
        :32A: 파싱
        형식: YYMMDDCCCAMOUNT
        예시: 240101KRW50000000,
        """
        pattern = re.compile(
            r'(?P<date>\d{6})'
            r'(?P<ccy>[A-Z]{3})'
            r'(?P<amount>[\d,]+)'
        )
        match = pattern.match(value.strip())
        
        if not match:
            return {}
        
        date_str = match.group("date")
        amount_str = match.group("amount").replace(",", ".")
        
        return {
            "value_date" : date(
                               int("20" + date_str[:2]),
                               int(date_str[2:4]),
                               int(date_str[4:6])
                           ),
            "currency"   : match.group("ccy"),
            "amount"     : Decimal(amount_str),
        }
    
    def _parse_customer_field(self, value: str) -> dict:
        """
        고객 필드 파싱 (:50K:, :59: 등)
        형식: /계좌번호\n이름\n주소
        """
        if not value:
            return {}
        
        lines = value.strip().split('\n')
        account = ""
        name    = ""
        address = []
        
        if lines and lines[0].startswith('/'):
            account = lines[0][1:].strip()
            lines   = lines[1:]
        
        if lines:
            name  = lines[0].strip()
            address = [l.strip() for l in lines[1:] if l.strip()]
        
        return {
            "account" : account,
            "name"    : name,
            "address" : " ".join(address),
        }
    
    def build_mt103_block4(self, data: MT103Data) -> str:
        """MT103 Block 4 생성"""
        
        # 날짜 포맷
        value_date_str = data.value_date.strftime("%y%m%d") if data.value_date else ""
        
        # 금액 포맷 (소수점 → 콤마)
        amount_str = str(data.amount).replace('.', ',')
        
        lines = ["{4:"]
        
        # 필수 태그
        lines.append(f":20:{data.sender_reference}")
        lines.append(f":23B:{data.bank_op_code}")
        lines.append(f":32A:{value_date_str}{data.currency}{amount_str}")
        
        # 송금의뢰인
        ordering = data.ordering_customer
        if ordering:
            lines.append(f":50K:/{ordering.get('account', '')}")
            lines.append(f"{ordering.get('name', '')}")
            if ordering.get('address'):
                lines.append(f"{ordering.get('address', '')}")
        
        # 선택 태그 (값 있을 때만)
        if data.ordering_institution:
            lines.append(f":52A:{data.ordering_institution}")
        if data.sender_correspondent:
            lines.append(f":53A:{data.sender_correspondent}")
        if data.receiver_corr:
            lines.append(f":54A:{data.receiver_corr}")
        if data.intermediary:
            lines.append(f":56A:{data.intermediary}")
        if data.account_with_inst:
            lines.append(f":57A:{data.account_with_inst}")
        
        # 수취인
        beneficiary = data.beneficiary
        if beneficiary:
            lines.append(f":59:/{beneficiary.get('account', '')}")
            lines.append(f"{beneficiary.get('name', '')}")
            if beneficiary.get('address'):
                lines.append(f"{beneficiary.get('address', '')}")
        
        # 부가 정보
        if data.remittance_info:
            lines.append(f":70:{data.remittance_info}")
        
        lines.append(f":71A:{data.charge_bearer}")
        
        if data.sender_to_receiver:
            lines.append(f":72:{data.sender_to_receiver}")
        if data.regulatory_reporting:
            lines.append(f":77B:{data.regulatory_reporting}")
        
        lines.append("-}")
        
        return "\n".join(lines)

## 6. Block 5 : Trailer Block

### 구조

```
{5:{CHK:ABCDEF123456}{TNG:}{PDE:1005120124WOOBURKRAXXX0000000001}}
    │                 │      │
    │                 │      └── {PDE:} Possible Duplicate Emission
    │                 └───────── {TNG:} Training 전문 표시
    └─────────────────────────── {CHK:} Checksum ★
```

### Block 5 주요 서브태그

```
┌──────────────────────────────────────────────────────────────────────┐
│  Block 5 태그 목록                                                     │
├────────┬──────────────────────────────────────────────────────────────┤
│  태그  │  설명                                                          │
├────────┼──────────────────────────────────────────────────────────────┤
│ {CHK:} │ Checksum (체크섬)                                             │
│        │ 12자리 16진수                                                  │
│        │ Block 1, 2, 4 내용 기반 계산                                   │
│        │ 전문 무결성 검증 핵심 ★                                        │
├────────┼──────────────────────────────────────────────────────────────┤
│ {TNG:} │ Training (훈련용 전문)                                         │
│        │ 테스트/훈련 목적 전문 표시                                       │
│        │ ⚠️ 실제 처리 금지                                              │
├────────┼──────────────────────────────────────────────────────────────┤
│ {PDE:} │ Possible Duplicate Emission (중복 발송 가능)                   │
│        │ 형식: HHMM + YYMMDD + BIC12 + Session + Sequence             │
│        │ 네트워크 오류 후 재발송 시 첨부                                  │
│        │ ★ 수신측 중복 처리 주의!                                        │
├────────┼──────────────────────────────────────────────────────────────┤
│ {PDM:} │ Possible Duplicate Message (중복 메시지 가능)                  │
│        │ 수신 전문에 첨부 (SWIFT 네트워크 감지)                           │
├────────┼──────────────────────────────────────────────────────────────┤
│ {SYS:} │ System Originated Message (시스템 발신)                        │
│        │ SWIFT 시스템이 자동 생성한 전문                                  │
├────────┼──────────────────────────────────────────────────────────────┤
│ {MRF:} │ Message Reference (메시지 참조)                                │
│        │ ACK/NAK 응신 전문에서 원본 참조                                 │
└────────┴──────────────────────────────────────────────────────────────┘
```

### PDE (Possible Duplicate Emission) 처리

```
PDE 발생 시나리오:

정상:
  우리은행 → [MT103 전송] → 한국은행
                  ↓
             한국은행 수신 완료

네트워크 오류 발생:
  우리은행 → [MT103 전송] → ??? (응답 없음)
              ↓ (타임아웃)
  우리은행 → [MT103 재전송 + {PDE:}] → 한국은행

한국은행 처리:
  1. {PDE:} 태그 감지
  2. 원본 Reference (:20:) 조회
  3. 이미 처리된 건이면 → 중복 처리 방지 (Idempotent)
  4. 미처리 건이면 → 정상 처리

⚠️  PDE 태그 없이 재발송 시 이중 처리 위험!
✅  실무에서 PDE 처리 로직 필수 구현
```

### Block 5 파싱 코드

```python
import re
from dataclasses import dataclass
from typing import Optional

@dataclass
class Block5:
    checksum             : Optional[str] = None  # {CHK:}
    is_training          : bool = False           # {TNG:}
    pde                  : Optional[str] = None  # {PDE:} 중복발송
    pdm                  : Optional[str] = None  # {PDM:} 중복메시지
    system_originated    : Optional[str] = None  # {SYS:}
    message_reference    : Optional[str] = None  # {MRF:}

class Block5Parser:
    """
    SWIFT Block 5 (Trailer) 파서
    """
    
    SUBTAG_PATTERN = re.compile(r'\{([A-Z]{3}):([^}]*)\}')
    BLOCK5_PATTERN = re.compile(r'\{5:(.*?)\}$', re.DOTALL)
    
    def parse(self, raw: str) -> Block5:
        """Block 5 파싱"""
        
        block5_match = self.BLOCK5_PATTERN.search(raw)
        if not block5_match:
            return Block5()
        
        content  = block5_match.group(1)
        raw_tags = {}
        
        for match in self.SUBTAG_PATTERN.finditer(content):
            raw_tags[match.group(1)] = match.group(2)
        
        return Block5(
            checksum          = raw_tags.get("CHK"),
            is_training       = "TNG" in raw_tags,
            pde               = raw_tags.get("PDE"),
            pdm               = raw_tags.get("PDM"),
            system_originated = raw_tags.get("SYS"),
            message_reference = raw_tags.get("MRF"),
        )
    
    def is_duplicate(self, block5: Block5) -> bool:
        """중복 전문 여부"""
        return block5.pde is not None or block5.pdm is not None
    
    def is_training_message(self, block5: Block5) -> bool:
        """훈련용 전문 여부 (실처리 금지)"""
        return block5.is_training
    
    def verify_checksum(self, 
                         block5    : Block5,
                         block1_raw: str,
                         block2_raw: str,
                         block4_raw: str) -> bool:
        """
        체크섬 검증
        실제 SWIFT 체크섬 알고리즘은 독자 규격
        여기서는 단순화된 버전 구현
        """
        if not block5.checksum:
            return False
        
        # 실제 구현 시 SWIFT 공식 알고리즘 적용 필요
        combined = block1_raw + block2_raw + block4_raw
        calculated = self._calculate_checksum(combined)
        
        return calculated == block5.checksum
    
    def _calculate_checksum(self, content: str) -> str:
        """
        SWIFT 체크섬 계산 (단순화 버전)
        실제: SWIFT 독자 CRC 알고리즘
        """
        import hashlib
        return hashlib.md5(content.encode()).hexdigest()[:12].upper()
    
    def build(self, 
              checksum  : Optional[str] = None,
              is_pde    : bool = False,
              pde_info  : str = "") -> str:
        """Block 5 생성"""
        
        tags = []
        
        if checksum:
            tags.append(f"{{CHK:{checksum}}}")
        
        if is_pde and pde_info:
            tags.append(f"{{PDE:{pde_info}}}")
        
        return "{5:" + "".join(tags) + "}"


# ── 사용 예시 ─────────────────────────────────────────────

parser = Block5Parser()

# 정상 전문
raw_normal = "{5:{CHK:ABCDEF123456}}"
b5_normal  = parser.parse(raw_normal)
print(f"체크섬        : {b5_normal.checksum}")      # ABCDEF123456
print(f"중복여부      : {parser.is_duplicate(b5_normal)}")  # False

# PDE 전문 (중복 가능)
raw_pde = "{5:{CHK:ABCDEF123456}{PDE:1005120124WOOBURKRAXXX0000000001}}"
b5_pde  = parser.parse(raw_pde)
print(f"중복여부      : {parser.is_duplicate(b5_pde)}")   # True
print(f"PDE 정보      : {b5_pde.pde}")

# 훈련용 전문
raw_tng = "{5:{TNG:}{CHK:ABCDEF123456}}"
b5_tng  = parser.parse(raw_tng)
print(f"훈련전문여부  : {parser.is_training_message(b5_tng)}")  # True
```

## 7. 전체 전문 통합 파서

```python
from dataclasses import dataclass
from typing import Optional

@dataclass
class SWIFTMessage:
    """SWIFT 전문 전체 도메인 객체"""
    block1  : Block1
    block2  : Block2Input | Block2Output
    block3  : Block3
    block4  : dict          # 태그 딕셔너리
    block5  : Block5
    mt_type : str           # "103", "202", "950" 등
    raw     : str           # 원본 전문

class SWIFTMessageParser:
    """
    SWIFT FIN 전문 통합 파서
    Block 1~5 전체 파싱
    """
    
    def __init__(self):
        self.b1_parser = Block1Parser()
        self.b2_parser = Block2Parser()
        self.b3_parser = Block3Parser()
        self.b4_parser = Block4Parser()
        self.b5_parser = Block5Parser()
    
    def parse(self, raw_message: str) -> SWIFTMessage:
        """SWIFT 전문 전체 파싱"""
        
        # 각 블록 파싱
        block1 = self.b1_parser.parse(raw_message)
        block2 = self.b2_parser.parse(raw_message)
        block3 = self.b3_parser.parse(raw_message)
        block4 = self.b4_parser.parse_block4(raw_message)
        block5 = self.b5_parser.parse(raw_message)
        
        # MT Type 추출
        mt_type = block2.mt_type
        
        return SWIFTMessage(
            block1  = block1,
            block2  = block2,
            block3  = block3,
            block4  = block4,
            block5  = block5,
            mt_type = mt_type,
            raw     = raw_message,
        )
    
    def parse_with_validation(self, 
                               raw_message: str) -> SWIFTMessage:
        """파싱 + 기본 검증"""
        
        msg = self.parse(raw_message)
        
        # 훈련용 전문 차단
        if self.b5_parser.is_training_message(msg.block5):
            raise ValueError("훈련용 전문입니다. 실처리 불가.")
        
        # 중복 전문 경고
        if self.b5_parser.is_duplicate(msg.block5):
            print(f"⚠️  중복 전문 감지: PDE={msg.block5.pde}")
        
        return msg
    
    def get_summary(self, msg: SWIFTMessage) -> dict:
        """전문 요약 정보"""
        
        return {
            "mt_type"       : msg.mt_type,
            "sender_bic"    : msg.block1.lt_address,
            "receiver_bic"  : getattr(msg.block2, 'receiver_bic', ''),
            "reference"     : msg.block4.get("20", ""),
            "amount_field"  : msg.block4.get("32A", ""),
            "uetr"          : msg.block3.uetr,
            "is_stp"        : self.b3_parser.is_stp(msg.block3),
            "is_gpi"        : self.b3_parser.is_gpi(msg.block3),
            "is_duplicate"  : self.b5_parser.is_duplicate(msg.block5),
            "checksum"      : msg.block5.checksum,
        }


# ── 전체 통합 사용 예시 ──────────────────────────────────

full_parser = SWIFTMessageParser()

raw_swift = """{1:F01WOOBURKRAXXX0000000000}
{2:I103HNBKKRSEXXX}
{3:{108:OWP20240101001}{119:STP}{121:a1b2c3d4-e5f6-7890-abcd-ef1234567890}}
{4:
:20:OWPREF240101001
:23B:CRED
:32A:240101KRW50000000,
:50K:/12345678901234
OVERSEAS COMPANY NEW YORK
:52A:CHASUS33
:57A:HNBKKRSE
:59:/98765432109876
KOREAN IMPORT COMPANY
:70:OFFSHORE KRW PAYMENT /INV/2024-001
:71A:SHA
-}
{5:{CHK:ABCDEF123456}}"""

# 파싱 실행
msg     = full_parser.parse_with_validation(raw_swift)

print("=" * 50)
print("SWIFT 전문 파싱 결과")

for key, value in summary.items():
    print(f"  {key:<16}: {value}")

## 8. Block별 실무 체크리스트

```
📋 Block 1 체크포인트
  □ Application ID = F (FIN) 확인
  □ Service ID = 01 확인
  □ LT Code 포함 여부 (12자리 LT Address)
  □ 발신 BIC = WOOBURKRAXXX 확인

📋 Block 2 체크포인트
  □ I (Input) / O (Output) 방향 확인
  □ MT Type 번호 정확성 (103/202/950 등)
  □ 수신 BIC = HNBKKRSEXXX 확인 (한국은행 발송 시)
  □ Priority 코드 (N/U/S)

📋 Block 3 체크포인트
  □ MUR ({108:}) = 내부 추적 Reference
  □ STP ({119:STP}) = 자동처리 가능 여부
  □ UETR ({121:}) = gpi 연동 시 필수
  □ Cover Payment ({103:COV}) 여부

📋 Block 4 체크포인트
  □ :20: Reference 형식 및 중복 여부
  □ :23B: CRED 확인
  □ :32A: 날짜/통화(KRW)/금액 정확성
  □ :50K: 송금의뢰인 형식 (STP 기준)
  □ :57A: HNBKKRSE (한국은행) 확인 ★
  □ :59: 수취인 계좌/이름 정확성
  □ :71A: SHA/OUR/BEN 수수료 코드
  □ :77B: 외환규제 보고 코드 (필요 시)

📋 Block 5 체크포인트
  □ {CHK:} 체크섬 12자리 확인
  □ {TNG:} 훈련전문 여부 (실처리 금지!)
  □ {PDE:} 중복발송 여부 → 멱등성 처리
  □ {PDM:} 중복메시지 여부 → 중복 확인
```
