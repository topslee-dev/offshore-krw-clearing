# 한국은행(BOK) 연동 규격

## 개요

역외원화결제 시스템은 한국은행의 SWIFT 게이트웨이를 통해 외부 거래와 연결됩니다.
본 문서는 한국은행 연동을 위한 메시지 규격과 처리 플로우를 설명합니다.

---

## 한국은행 BIC

| 항목 | 값 |
|------|-----|
| **BIC** | HNBKKRSEXXX |
| **정식 명칭** | Bank of Korea |
| **SWIFT 주소** | HNBKKRSE |
| **위치** | 서울 |

---

## 연동 메시지 유형

### 수신 메시지 (우리은행 → BOK)

| 메시지 | 설명 | 용도 |
|--------|------|------|
| MT103 | 송금 통보 | 역외원화 결제 시 한국은행 보고 |
| MT202 | 자금이체 | 은행간 대금 결제 |
| MT300 | 환율 거래 | FX 계약 보고 (선택) |

### 발신 메시지 (BOK → 우리은행)

| 메시지 | 설명 | 용도 |
|--------|------|------|
| MT900 | 차변 확인 | 계좌 차감 통보 |
| MT910 | 대변 확인 | 계좌 입금 통보 |
| MT940 | 잔액 보고 | 일일 노스트로 잔액 보고 |
| MT950 | 은행 잔액 보고 | 은행 간 정산 |

## 메시지 플로우

### 일반 역외원화결제 플로우

```
우리은행(BOK-Lite)          한국은행              해외은행
      │                       │                    │
      │────MT103 송금─────────>│                    │
      │                       │                    │
      │<───MT900 확인─────────│                    │
      │                       │                    │
      │<───MT910 수신확인──────│                    │
      │                       │                    │
      │────MT202 자금이체────>│                    │
      │                       │                    │
      │<───MT950 잔액보고──────│                    │
```

### 단계별 상세

#### 1단계: MT103 송금 통보

우리은행이 한국은행에 역외원화 결제 건을 보고합니다.

```
발신: WOOBURKRSAXXX (우리은행)
수신: HNBKKRSEXXX (한국은행)
유형: MT103
```

**주요 태그:**
- :20: - 참조번호
- :32A: - 금액 (KRW)
- :50K: - 송금 고객
- :59: - 수령 고객
- :57A: - 수령 은행 (우리은행 BIC)

#### 2단계: MT900 차변 확인

한국은행이 우리은행 계좌에서 해당 금액을 차감합니다.

```
발신: HNBKKRSEXXX (한국은행)
수신: WOOBURKRSAXXX (우리은행)
유형: MT900
```

#### 3단계: MT910 대변 확인

우리은행이 수신 확인 응답을 보냅니다.

```
발신: WOOBURKRSAXXX (우리은행)
수신: HNBKKRSEXXX (한국은행)
유형: MT910
```

#### 4단계: MT202 자금이체

실제 자금이체가 발생합니다.

```
발신: WOOBURKRSAXXX (우리은행)
수신: HNBKKRSEXXX (한국은행)
유형: MT202
```

#### 5단계: MT950 잔액 보고

한국은행이 일일 nostro 계좌 잔액을 보고합니다.

```
발신: HNBKKRSEXXX (한국은행)
수신: WOOBURKRSAXXX (우리은행)
유형: MT950
```

## 우리은행 전용 필드 설정

### Block 1 (Basic Header)

```
{F01WOOBURKRSAXXX0000000000}
```

- BIC: WOOBURKRSAXXX (우리은행)
- Sequence: 0000000000

### Block 2 (Application Header)

```
{I103HNBKKRSEXXX}  (수신 시)
{O103HNBKKRSEXXX}  (송신 시)
```

- I: Input (수신)
- O: Output (송신)

## 원화(KRW) 처리 규칙

### 금액 형식

- 원화(KRW)는 소수점 이하 없음
- 예: 50,000,000 KRW → `50000000`
- 3자리마다 콤마 표기 (사람용)

### Value Date

- 역외원화 결제: 당일 Valor
- 원화 국내 결제: 당일 Valor (16:30 이전)
- 마감 이후: 다음 영업일

### 원화 계좌 번호

- 10자리 숫자
- 예: /1234567890

### 역외원화 환전 유형

| 유형 | 설명 | 비고 |

| EUR/KRW | 유로-원화 | 비교적 적음 |

| 구분 | 적용처 | 스프레드 |

| 기준환율 | 일반 거래 | 시장 스프레드 |
| 재정환율 | 대금 결제 | 협정 스프레드 |

## 예외 시나리오

### DUPLICATE_REF (중복 참조)

동일 참조번호로 중복 전송 시:

1. 첫 번째 메시지: 정상 처리
2. 두 번째 메시지: `ACK299` 응답 (DUPLICATE)
3. 자동 Reprocessing 방지

### INVALID_BIC (잘못된 BIC)

BIC 코드 검증 실패 시:

1. `NAK199` 오류 응답
2. Repair Queue 등록
3. 수동 수정 후 재처리

### AMOUNT_MISMATCH (금액 불일치)

MT103 vs MT202 금액 불일치 시:

1. MT103 금액 우선
2. MT202 금액 조정
3. 정산 부서 Escalation

### BOK_TIMEOUT (응답 타임아웃)

30초 이내 응답 없음:

1. 재시도 (1회)
2. 대기열 등록
3. 수동 추적

## 보고 기준 금액 산출

한국은행 보고용으로 다음 금액을 산출합니다:

```python
def calculate_bok_reporting_amount(amount: Decimal, currency: str) -> Decimal:
    """
    원화 기준:
    - 정확한 금액 그대로
    - 소수점 이하 반올림

    외화 기준:
    - 재정환율 적용
    - 소수점 이하 반올림
    """
    if currency == "KRW":
        return amount.quantize(Decimal("1"))
    else:
        # 재정환율 적용
        rate = get_financial_rate(currency)
        krw_amount = amount * rate
        return krw_amount.quantize(Decimal("1"))
```

## 상태 코드

### BOK 응답 코드

| 코드 | 의미 | 조치 |
|------|------|------|
| ACK199 | 정상 수신 | 처리 진행 |
| ACK299 | 중복 수신 | 무시 |
| NAK199 | 형식 오류 | Repair 필요 |
| NAK299 | 금액 불일치 | 정산 확인 |

### 내부 상태

| 상태 | 설명 |
|------|------|
| RECEIVED | 전문 수신 |
| VALIDATED | 검증 완료 |
| PROCESSING | 처리 중 |
| PENDING_BOK | BOK 응답 대기 |
| SETTLED | 정산 완료 |
| ERROR | 오류 발생 |
| REPAIR | 수동 복구 대기 |
